# 1.6. Translation Instructions

## Cloning the repository

* Clone the [**wlc-engine-translate**](https://wlcgitlab.egamings.com/wlc/wlc-engine-translate.git) repository to the **Projects/src** directory. To do this, run the command:\
**git clone git@wlcgitlab.egamings.com:wlc/wlc-engine-translate.git**
* Go to **wlc-engine-translate** and set the dependencies:\
**cd wlc-engine-translate**\
**npm i**

## Preparing to add translations

If the repository is already cloned, the update of the local repository status is needed before adding new translations.

* In the **Projects/src/wlc-engine-translate** directory, switch to the **master** branch, the command: **git checkout master**.
* Pull the actual changes: **git pull origin master**.
* Before adding new translations, create and move to a new branch: **git checkout -b scrXXXXXX** (the branch number is the same as the task number).

## Adding new translations

* To add all phrases from **gettext** and **translate**, run the command: **gulp engineMessages** in the docker-container
**wlc-engine**.
* After successful command execution, check in the ***.po** file in the **wlc-engine-translate** directory whether the necessary phrase has appeared.
* Add translations to the required language files.
* If translations need to be added manually, then it is mandatory to check whether the phrase is not duplicated.\
Check with the command: **find -name "\*.po" | while read file; do msgfmt --check "$file"; done**.
* Make a commit, the command: **git commit -am "SCR #XXXXXX - add translations"**.
* Push all changes to the remote repository, the command: **git push origin scrXXXXXX**.\
* After successfully pushing the changes, create an MR and attach it to the task.

## Translations testing

* To test translations, the **wlc-engine-translate** branch should be attached in the project (for example: devcasino), and not in the engine.
* To attach the translations, add **"@egamings/wlc-engine-translate": "git+ssh://git@wlcgitlab.egamings.com:wlc/wlc-engine-translate.git#scrXXXXXX"** in the **package.json** file to the "dependencies" array.
* Create a branch in a project with attached translations.

# 1.6. Репозиторий переводов "wlc-engine-translate".

## Клонирование репозитория.

* В директорию **Projects/src** склонировать репозиторий [**wlc-engine-translate**](https://wlcgitlab.egamings.com/wlc/wlc-engine-translate.git). Для этого выполнить команду:\
**git clone git@wlcgitlab.egamings.com:wlc/wlc-engine-translate.git**
* Далее необходимо перейти в **wlc-engine-translate** и установить зависимости:\
**cd wlc-engine-translate**\
**npm i**

## Подготовка к добавлению переводов.

Если репозиторий уже склонирован, то перед добавлением новых переводов необходимо актуализировать состояние локального репозитория.

* Находясь в директории **Projects/src/wlc-engine-translate** переключиться на ветку **master**, команда: **git checkout master**.
* Далее стянуть актуальные изменения: **git pull origin master**.
* Перед добавлением новых переводов создать и перейти на новую ветку: **git checkout -b scrXXXXXX** (номер ветки такой же как и номер задачи).

## Добавление новых переводов.

* Для добавления всех фраз из **gettext** и **translate** необходимо выполнить команду: **gulp engineMessages**, находясь в docker-контейнере
**wlc-engine**.
* После успешного выполнения команды проверить в ***.po** файле в директории **wlc-engine-translate** появилась ли нужная фраза.
* Добавить переводы в необходимые языковые файлы.
* Если переводы нужно добавить вручную, то в обязательном порядке проверить не дублируется ли добавляемая вами фраза.\
Выполнить проверку можно командой: **find -name "\*.po" | while read file; do msgfmt --check "$file"; done**.
* Далее делаем коммит, команда: **git commit -am "SCR #XXXXXX - add translations"**.
* Делаем пуш всех изменений на удаленный репозиторий, команда: **git push origin scrXXXXXX**.\
Хук **pre-push** дополнительно проверит файлы на ошибки и ели такие есть выведет в консоль список фалов в которых они есть.
* После успешного пуша изменений создать МР и прикрепить к задаче.

## Тестирование переводов.

* Для тестирования переводов ветка **wlc-engine-translate** должна быть залочена в проекте (например: devcasino), а не в движке. 
* Чтобы залочить переводы нужно в файле **package.json** в массив "dependencies" добавить:\
**"@egamings/wlc-engine-translate": "git+ssh://git@wlcgitlab.egamings.com:wlc/wlc-engine-translate.git#scrXXXXXX"**
* Создать ветку в проекте с залоченными переводами.
