# 1.6. Подмодуль языков "Languages".

## Инициализация подмодуля.

* Для инициализации подмодуля языков, необходимо выполнить следующую команду **git submodule update --init** находясь в
  директории **wlc-engine**.
* Если подмодуль не удалось инициализировать, необходимо удалить все содержимое в папке languages и повторить команду
  **git submodule update --init**.

При успешной инициализации подмодуля должно появится следующие сообщение:
**Подмодуль по пути «./»: забрано состояние «XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX» (хэш будет отличаться).**

## Подготовка к добавлению переводов.

Если подмодуль уже существует, то перед добавлением новых переводов необходимо актуализировать состояние подмодуля.

* Для этого необходимо переключится на ветку **master** находясь в директории **wlc-engine/src/system/languages**.
  Выполнив команду **git checkout master**.
* Далее необходимо сделать пул мастера - команда **git pull origin master**.
* Переключаемся на новую ветку для добавления новых переводов, команда - **git checkout -b scrXXXXXX** (номер ветки
  такой же как и в основной задаче для **wlc-engine**)

## Добавление новых переводов.

* Для добавления всех фраз из **gettext** и **translate** необходимо выполнить команду - **gulp engineMessages**
  находясь в **docker-контейнере wlc-engine**.
* После успешного добавления новых фраз ищем файл **ru.po** в папке **languages**, проверяем появилась ли нужная фраза и
  добавляем к ней перевод.
* Если переводы добавляем в ручную, то в обязательном порядке необходимо проверить не дублируется ли добавляемая вами
  фраза. Выполнить проверку можно командой:  **find -name "\*.po" | while read file; do msgfmt --check "$file"; done**

## Добавление изменений на удаленный репозиторий.

После того как добавили необходимые переводы и убедились в том, что дубли отсутствуют, делаем коммит и пуш всех
изменений на удаленный репозиторий.

* Для этого выполним команду **git commit -am 'SCR #XXXXXX - add translation"**.
* Далее делаем пуш **git push origin scrXXXXXX**.
* После успешного пуша подтверждаем МР и прикрепляем к задаче.

## Удаляем подмодуль из МРа основной задачи.

Если вы создали МР основной задачи и туда “случайным образом” попал подмодуль и его необходимо убрать, делаем следующее:

* Находясь в директории **wlc-engine/src/system/languages** переключаемся на актуальный мастер (делаем пул мастера).
* Далее необходимо в основном репозитории (**wlc-engine**) добавить эти изменения и запушить. Изменения в МРе что
  подбмодуль был изменен должны исчезнуть.
* После принятия МРа с переводами в директории **wlc-engine/src/system/languages** необходимо переключится на актуальный
  мастер (сделать пул).
* При необходимости пересобрать проект, команда - **npm run dev**.
