# 1.6. The Languages submodule

## Initialization of the submodule

* To initialize the Languages submodule, run the following command from the **wlc-engine** directory: **wlc-engine**.
* f the submodule fails to initialize, then empty the "languages" folder and run the following command again:
  **git submodule update --init**.

f the submodule has been initialized successfully, then the following message will appear: **Submodule in directory "./": the current state is XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX** (the hash value will differ).

## Preparation of the translations

If the submodule already exists, make sure to update its state before adding new translations.

* To do so, switch to the master branch from the **wlc-engine/src/system/languages** directory by running the following command: **git checkout master**.
* Then, pull the master branch by running the following command: **git pull origin master**
* Switch to the new branch for adding new translations by running **git checkout -b scrXXXXXX** (the branch number is the same as in the core task for **wlc-engine**)

## Adding new translations

* To add all phrases from **gettext** and **translate**, run the following command from the **wlc-engine** docker container: **gulp engineMessages**
* Once the new phrases are successfully added, search for the file called **ru.po** in the **languages** folder, make sure that the necessary phrase is present and then add its translation.
* When adding the translations manually, always check that the phrase you're going to add is not a duplicate of a previously added phrase. Run the following command to check the results: **find -name "\*.po" | while read file; do msgfmt --check "$file"; done**

## Adding changes to a remote repository

Once you've added the necessary translations and checked them for duplicates, commit and push all changes to the remote repository.

* To do so, run **git commit -am 'SCR #XXXXXX - add translation"**.
* Then, use the push command: **git push origin scrXXXXXX**.
* After the successful pushing, approve the MP and attach it to the task.

## Removing the submodule from the MP of the core task

If you have created an MP of the core task and pushed there the submodule by mistake, perform the following:

* From the **wlc-engine/src/system/languages** directory, switch to the actual master branch (perform the master pull operation).
* Then, add these changes to the main repository (**wlc-engine**) and push them. Changes to the MP caused by changes to the submodule will disappear.
* After accepting the MP with translations in the **wlc-engine/src/system/languages** directory, switch to the actual master branch (perform the pull operation).
* Rebuild the project when necessary by running the following command - **npm run dev**.

# 1.6. Подмодуль языков "Languages".

## Инициализация подмодуля.

* Для инициализации подмодуля языков, необходимо выполнить следующую команду **git submodule update --init** находясь в
  директории **wlc-engine**.
* Если подмодуль не удалось инициализировать, необходимо удалить все содержимое в папке languages и повторить команду
  **git submodule update --init**.

При успешной инициализации подмодуля должно появится следующие сообщение:
**Подмодуль по пути «./»: забрано состояние «XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX» (хэш будет отличаться).**

## Подготовка к добавлению переводов.

Если подмодуль уже существует, то перед добавлением новых переводов необходимо актуализировать состояние подмодуля.

* Для этого необходимо переключится на ветку **master** находясь в директории **wlc-engine/src/system/languages**.
  Выполнив команду **git checkout master**.
* Далее необходимо сделать пул мастера - команда **git pull origin master**.
* Переключаемся на новую ветку для добавления новых переводов, команда - **git checkout -b scrXXXXXX** (номер ветки
  такой же как и в основной задаче для **wlc-engine**)

## Добавление новых переводов.

* Для добавления всех фраз из **gettext** и **translate** необходимо выполнить команду - **gulp engineMessages**
  находясь в **docker-контейнере wlc-engine**.
* После успешного добавления новых фраз ищем файл **ru.po** в папке **languages**, проверяем появилась ли нужная фраза и
  добавляем к ней перевод.
* Если переводы добавляем в ручную, то в обязательном порядке необходимо проверить не дублируется ли добавляемая вами
  фраза. Выполнить проверку можно командой:  **find -name "\*.po" | while read file; do msgfmt --check "$file"; done**

## Добавление изменений на удаленный репозиторий.

После того как добавили необходимые переводы и убедились в том, что дубли отсутствуют, делаем коммит и пуш всех
изменений на удаленный репозиторий.

* Для этого выполним команду **git commit -am 'SCR #XXXXXX - add translation"**.
* Далее делаем пуш **git push origin scrXXXXXX**.
* После успешного пуша подтверждаем МР и прикрепляем к задаче.

## Удаляем подмодуль из МРа основной задачи.

Если вы создали МР основной задачи и туда “случайным образом” попал подмодуль и его необходимо убрать, делаем следующее:

* Находясь в директории **wlc-engine/src/system/languages** переключаемся на актуальный мастер (делаем пул мастера).
* Далее необходимо в основном репозитории (**wlc-engine**) добавить эти изменения и запушить. Изменения в МРе что
  подбмодуль был изменен должны исчезнуть.
* После принятия МРа с переводами в директории **wlc-engine/src/system/languages** необходимо переключится на актуальный
  мастер (сделать пул).
* При необходимости пересобрать проект, команда - **npm run dev**.
