# 1.5. Сборка

## Описание

WLC-Engine представляет собой библиотеку модулей Angular, а так же файлы конфигурации Gulp, TypeScript и Webpack (
дополнительные настройки для сборщика Angular). Данная библиотека представляет собой полностью готовый, с точки зрения
кода, продукт, но без необходимых для самостоятельного запуска файлов конфигурации.

## Порядок сборки проекта

Ниже представлен порядок сборки проекта на WLC-Engine, созданного на основе заранее созданного шаблона (npm и composer
зависимости установлены, docker контейнеры запущены):

1. Сборка осуществляется командой npm **run dist**. По этой команде запускается таск-менеджер **gulp** с задачей **
   dist**.
2. Локальный **gulpfile.js** импортирует файл движка по пути **./node_modules/@egamings/wlc-engine/gulpfile/gulpfile** и
   запускает соответствующий таск.
3. После подготовки сборки (см раздел gulp) gulp запускает основную сборку проекта, используя стандартный инструмент
   **Angular-cli: ng build**.
4. Сборщик подключает локальный (проектный) файл **./tsconfig.app.json**, который наследует параметры от файла движка
   **./node_modules/@egamings/wlc-engine/tsconfig.app.json**.
5. Все настройки Angular располагаются в локальном файле **./angular.json** и не наследуются (Angular не поддерживает
   наследования файла конфигурации).
6. Индексный файл, с которого начинается сборка проекта, располагается в по пути **./src/main.ts**.
7. Данный файл импортирует основной модуль **App** по пути **./wlc-engine/modules/app/app.module**, при этом **
   wlc-engine** - алиас, указанный в файле **./tsconfig.app.json**.
8. Символическая ссылка на директорию **./wlc-engine** формируется автоматически в **gulp** перед запуском сборки и
   указывает в директорию **./node_modules/@egamings/wlc-engine/src**.
9. Далее происходит сборка проекта так, как будто все файлы движка располагаются непосредственно в проекте **WLC**, а не
   в
   **./node_modules**. Это необходимо для корректной сборки проекта.
10. В процессе сборки так же используются файлы конфигурации, располагаемые по пути **./config/frontend/*.ts**. Данные
    файлы содержат необходимые настройки проекта. Их подробная структура рассмотрена в разделе "WLC Engine -
    конфигурация".

## Особенности сборки

### Подготовительный этап

Перед запуском сборщика Angular выполняются дополнительные задачи gulp, которые подготавливают сборку:

1. Создаётся или очищается директория **dist**.
2. Создаётся директория для хранения временных файлов.
3. Создаётся симлинк на индексный файл **index.html**, необходимый для корректной работы **wlc-core**.
4. Создаётся симлинк на директорию **wlc-engine** в корне проекта, необходимый для корректной сборки.
5. Создаются дополнительный симлинки на файлы, требующиеся во время сборки.
6. Запускается задача по формированию JSON файлов локализации проекта на основе **PO** файлов. Для этого используется **
   GNU linux**, поэтому нативная сборка на Windows невозможна.
7. Запускается команда **npm run build:dev** или **npm run build:prod** непосредственно через оболочку shell, при этом
   процесс gulp не останавливается (для dev сборки), хотя сама сборка происходит в параллельном процессе. Это необходимо
   для корректной работы watch режима.

### Этап сборки

Сборка проекта происходит стандартными средствами Angular. Особенностью является только дополнительные параметры
Webpack, необходимые для формирования файловой структуры непосредственно в проекте. (см. раздел WLC Engine -
кастомизация).

Файл конфигурации сборки **angular.json** расположен в корневой директории WLC проекта.

## Результат сборки

Результатом является набор готовых бандлов в директории **./roots/static/dist** и симлинк на **index.html** файл.
