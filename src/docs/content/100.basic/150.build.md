# 1.5. Assembly

## Description

WLC-Engine is a library of Angular modules along with the configuration files of Gulp, TypeScript and Webpack (additional settings for the Angular builder). This library is a ready-to-use product in terms of the code base, but without the configuration files required for the self-launching.

## The project assembly procedure

The assembly procedure for a WLC-Engine project composed from a previously created template (provided that "npm" and "composer" dependencies are installed and docker containers are up and running) is as follows:

1. Use the following command to assemble your project: **npm run dist**. This will launch the task manager **gulp** with the task **dist**.
2. Local **gulpfile.js** imports the engine file from **./node_modules/@egamings/wlc-engine/gulpfile/gulpfile** and then runs the corresponding task.
3. After the pre-assembly (refer to the section about gulp), gulp starts the main assembly process for the project using the standard tool Angular-cli: **ng build**.
4. The builder includes local (project-wide) file **./tsconfig.app.json**, which inherits parameters from engine file **./node_modules/@egamings/wlc-engine/tsconfig.app.json**.
5. All Angular settings are stored in the local file **./angular.json** and are not inherited (Angular doesn't support inheritance of configuration files).
6. The indexing file which is used to start the project assembly is stored in **./src/main.ts**.
7. This file imports core module App from **./wlc-engine/modules/app/app.module**, where **wlc-engine** is the alias indicated in file **./tsconfig.app.json**.
8. The symbolic link to directory **./wlc-engine** is generated automatically in gulp before the assembly and points to directory **./node_modules/@egamings/wlc-engine/src**.
9. Then the project is assembled as if all engine files were located directly in the **WLC** project rather than in **./node_modules**. This is required for the proper project assembly.
10. During the assembly process, the engine also uses configuration files from **./config/frontend/*.ts**. These files contain the required project settings. For more details on their structure, please refer to "Configuration of WLC Engine".

## Assembly features

### Pre-assembly

Before the Angular builder starts, gulp performs the following pre-assembly tasks:

1. Creates or empties the **dist** directory.
2. Creates a new directory for temporary files.
3. Creates a symlink to indexing file **index.html** which is required for the proper operation of **wlc-core**.
4. Creates a symlink to directory **wlc-engine** in the project root folder for the proper assembly.
5. Creates additional symlinks to the files required for the assembly.
6. Launches the generation of JSON localization files based on **PO** files. Since this task uses **GNU linux**, so native assembly on Windows is not supported.
7. The engine runs command **npm run build:dev** or **npm run build:prod** directly in the shell, and the gulp process doesn't stop (for the dev assembly) even despite the fact that the assembly process is running in parallel. This is necessary for the proper operation of the watch mode.

### Assembly

Projects are assembled using standard Angular tools. The only special feature is that additional Webpack parameters are required to generate the file structure directly in the project (for more details, please refer to [Customization of WLC Engine]()).

The configuration file of assembly **angular.json** is stored in the root directory of your WLC project.

## Results of the assembly

The results include a set of ready-to-use bundles in directory **./roots/static/dist** and a symlink to file **index.html**.

# 1.5. Сборка

## Описание

WLC-Engine представляет собой библиотеку модулей Angular, а так же файлы конфигурации Gulp, TypeScript и Webpack (
дополнительные настройки для сборщика Angular). Данная библиотека представляет собой полностью готовый, с точки зрения
кода, продукт, но без необходимых для самостоятельного запуска файлов конфигурации.

## Порядок сборки проекта

Ниже представлен порядок сборки проекта на WLC-Engine, созданного на основе заранее созданного шаблона (npm и composer
зависимости установлены, docker контейнеры запущены):

1. Сборка осуществляется командой npm **run dist**. По этой команде запускается таск-менеджер **gulp** с задачей **dist**.
2. Локальный **gulpfile.js** импортирует файл движка по пути **./node_modules/@egamings/wlc-engine/gulpfile/gulpfile** и
   запускает соответствующий таск.
3. После подготовки сборки (см раздел gulp) gulp запускает основную сборку проекта, используя стандартный инструмент
   **Angular-cli: ng build**.
4. Сборщик подключает локальный (проектный) файл **./tsconfig.app.json**, который наследует параметры от файла движка
   **./node_modules/@egamings/wlc-engine/tsconfig.app.json**.
5. Все настройки Angular располагаются в локальном файле **./angular.json** и не наследуются (Angular не поддерживает
   наследования файла конфигурации).
6. Индексный файл, с которого начинается сборка проекта, располагается в по пути **./src/main.ts**.
7. Данный файл импортирует основной модуль **App** по пути **./wlc-engine/modules/app/app.module**, при этом **wlc-engine** - алиас, указанный в файле **./tsconfig.app.json**.
8. Символическая ссылка на директорию **./wlc-engine** формируется автоматически в **gulp** перед запуском сборки и
   указывает в директорию **./node_modules/@egamings/wlc-engine/src**.
9. Далее происходит сборка проекта так, как будто все файлы движка располагаются непосредственно в проекте **WLC**, а не
   в
   **./node_modules**. Это необходимо для корректной сборки проекта.
10. В процессе сборки так же используются файлы конфигурации, располагаемые по пути **./config/frontend/*.ts**. Данные
    файлы содержат необходимые настройки проекта. Их подробная структура рассмотрена в разделе "WLC Engine -
    конфигурация".

## Особенности сборки

### Подготовительный этап

Перед запуском сборщика Angular выполняются дополнительные задачи gulp, которые подготавливают сборку:

1. Создаётся или очищается директория **dist**.
2. Создаётся директория для хранения временных файлов.
3. Создаётся симлинк на индексный файл **index.html**, необходимый для корректной работы **wlc-core**.
4. Создаётся симлинк на директорию **wlc-engine** в корне проекта, необходимый для корректной сборки.
5. Создаются дополнительный симлинки на файлы, требующиеся во время сборки.
6. Запускается задача по формированию JSON файлов локализации проекта на основе **PO** файлов. Для этого используется **GNU linux**, поэтому нативная сборка на Windows невозможна.
7. Запускается команда **npm run build:dev** или **npm run build:prod** непосредственно через оболочку shell, при этом
   процесс gulp не останавливается (для dev сборки), хотя сама сборка происходит в параллельном процессе. Это необходимо
   для корректной работы watch режима.

### Этап сборки

Сборка проекта происходит стандартными средствами Angular. Особенностью является только дополнительные параметры
Webpack, необходимые для формирования файловой структуры непосредственно в проекте. (см. раздел WLC Engine -
кастомизация).

Файл конфигурации сборки **angular.json** расположен в корневой директории WLC проекта.

## Результат сборки

Результатом является набор готовых бандлов в директории **./roots/static/dist** и симлинк на **index.html** файл.
