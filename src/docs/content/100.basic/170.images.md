# 1.7. Работа с изображениями в проекте

Изображения в проекте могут быть доступны из трех источников:

* через конфиг files (папка src/svg),
* через папку root/static/images,
* через сервер статики gstatic.

## Конфиг files

Конфиг files служит для добавления в проект и движок изображений в формате SVG. В проекте: **
config/frontend/03.files.config.ts**
В движке: **src/modules/core/system/config/files.config.ts**

Важно, через конфиг files подключаются только SVG-изображения, которые вставляются в шаблон инлайном, т.е. не теряют
векторную структуру. Также файлы, вставленные через конфиг files включаются непосредственно в бандл сборки. Таким
образом, для отображения картинки в рантайме (в браузере) не требуется делать отдельный сетевой запрос, что увеличивает
скорость загрузки страницы. Данный способ подключения изображений следует использовать для небольших изображений,
например иконок (до 5кб), это позволит ускорить загрузку страниц, но незначительно увеличит размер бандла.

Чтобы получить SVG-разметку нужно:

1) Задекларировать файл в конфиге проекта или движка в формате

```typescript
KEY: PATH
```

где PATH - путь к файлу относительно папки src/svg (например - 'close': 'icons/close.svg').

2) Положить файл по соответствующему пути в src/svg.
3) Вызвать метод FilesService.getSvgByName(KEY). Файлы могут храниться как в проекте, так и в движке. Файл следует
   декларировать там где он лежит. Файлы в проекте имеют больший приоритет, при совпадении имен в проекте и движке будет
   взять файл из проекта.

Метод FilesService.getSvgByName всегда возвращает объект вида:

```typescript
{
    key: string; // переданный ключ
    location:   'local-wlc' | 'local-engine'; // фактическое размещение файла
    htmlString: string; // разметка SVG
}
```

Если файл по переданному ключу не найден, объект будет содержать только поле key.

## Папка root/statiс/images и сервер статики gstatic.

Папка root/statiс/images используется для хранения изображений проекта. Может хранить любые форматы. Файлы из этой папки
не попадают в бандл. Бинарные файлы (jpj, png, gif, webp) загружаются отдельными запросам через атрибут SRC тега IMG.
Файлы SVG, также загружаются отдельным запросом, но могут вставляться в шаблон инлайном, как SVG-разметка.

Чтобы получить файл, нужно вызвать метод FilesService.getFileByUrl(URL), где URL - путь до файла относительно папки
root/statiс/images

Например файл лежит по пути **roots/static/images/icons/sidebar-link-arrow.svg** значит URL должен быть
**icons/sidebar-link-arrow.svg**.

Метод вернет объект вида:

```typescript
{
    key: string; // переданный ключ(url)
    location:   'static' | 'gstatic'; // фактическое размещение файла
    url: string; // относительный путь до изображения
    htmlString: string; // разметка SVG, если запрошенный файл имеет расширение SVG (опционально)
}
```

Файлы могут быть размещены как локально в папке **root/statiс/images**, так и на сервере статики **gstatic**. Метод сам
определит расположение файла и вернет нужный путь. В первую очередь метод проверит **root/statiс/images**.

Важно! В случаях когда метод возвращает путь на сервер **gstatic**, наличие нужного файла не гарантируется и не
проверяется.

## Встраивание изображений в шаблон.

Для непосредственной вставки изображения в шаблон следует использовать компонент **wlc-icon**

Вызывается как атрибут. Принимает параметры:

* **iconUrl** - путь до изображения, аналогичен параметру URL метода **FilesService.getFileByUrl**.
* **iconName** - ключ, указанный в конфиге files, аналогичен параметру **KEY** метода **FilesService.getSvgByName**.

Приоритет у параметра **iconUrl**.

Если целевой файл является бинарным, компонент вставит в шаблон тег IMG с атрибутом **SRC=<_путь до файла_>**.

Если целевой файл SVG, компонент вставит в шаблон разметку SVG.
