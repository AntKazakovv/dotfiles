# 1.7. Working with images in projects

Images can be included in projects from three sources:

* The "files" configuration file (src/svg)
* The folder at root/static/images
* The "gstatic" static files server

## The "files" configuration file

The "files" configuration file allows you to add SVG images to your projects or to the engine itself. In the project: **config/frontend/03.files.config.ts** In the engine: **src/modules/core/system/config/files.config.ts**

Note that only inline SVG images (that keep their vector-based structure) can be included using the "files" configuration file. All files included via the "files" configuration file are added directly to the assembly bundle. Thus, displaying an image in the runtime (i.e. in a browser) won't require a separate network request, and the web page will load faster. Use this method for small images, such as icons (up to 5 kb), to increase the page load speed while keeping the bundle relatively small.

To obtain the SVG layout, perform the following:

1) Declare the image file in the project or engine configuration file in the following format:

```typescript
KEY: PATH
```

where PATH is the path to the file relative to src/svg (e.g. 'close': '**icons/close.svg**').

2) Move the file to the corresponding folder in src/svg.
3) Call the `FilesService.getSvgByName`(`KEY`) method. Files can be stored in a project or in the engine. Make sure to declare each file in the folder where it is actually located. Files in projects have a higher priority, so if any two files have the same names in the project and in the engine, then the file from the project will be included.

The `FilesService.getSvgByName` method always returns a object with the following structure:

```typescript
{
    key: string; // переданный ключ
    location:   'local-wlc' | 'local-engine'; // фактическое размещение файла
    htmlString: string; // разметка SVG
}
```

If no file matching the key was found, then the object will contain only the key field.

## The root/statiс/images folder and the "gstatic" static files server

The **root/statiс/images** folder stores the images used in your project. It supports all image types. Files from this folder won't be added to the bundle. Binary files (jpj, png, gif, webp) are included using separate queries via the `SRC` attribute of the `IMG` tag. SVG files are also included by separate queries, but can be inserted inline into the template similar to SVG layouts.

To obtain a file, call the `FilesService.getFileByUrl`(`URL`) method, where URL is the path to the desired file relative to **root/statiс/images**

For example, if the required file is located in **roots/static/images/icons/sidebar-link-arrow.svg** then its URL is
**icons/sidebar-link-arrow.svg**.

The method will return an object with the following structure:

```typescript
{
    key: string; // переданный ключ(url)
    location:   'static' | 'gstatic'; // фактическое размещение файла
    url: string; // относительный путь до изображения
    htmlString: string; // разметка SVG, если запрошенный файл имеет расширение SVG (опционально)
}
```

Files can be located either locally in **root/statiс/images** or on the static files server called **gstatic**. The method will find the file and return the desired path. First of all, the method will search for the file in **root/statiс/images**.

**Important!** When the method returns a path to the **gstatic** server, it means that the desired file may be missing.

## Embedding images in a template

To insert an image directory into a template, use the **wlc-icon** component

It is called as an attribute. The input parameters are as follows:

* `iconUrl` - is the path to the image, which is similar to the URL parameter of the `FilesService.getFileByUrl`
* `iconName` - is the key indicated in the "files" configuration file, which is similar to the `KEY` parameter of the `FilesService.getSvgByName` method

The `iconUrl` parameter has a higher priority.

If the target file is a binary one, then the component will insert the IMG tag with the `SRC=<_path_to_file_>` attribute into the template.

If the target is an SVG file, then the component will insert the SVG layout into the template.

# 1.7. Работа с изображениями в проекте

Изображения в проекте могут быть доступны из трех источников:

* через конфиг files (папка src/svg),
* через папку root/static/images,
* через сервер статики gstatic.

## Конфиг files

Конфиг files служит для добавления в проект и движок изображений в формате SVG. В проекте: **
config/frontend/03.files.config.ts**
В движке: **src/modules/core/system/config/files.config.ts**

Важно, через конфиг files подключаются только SVG-изображения, которые вставляются в шаблон инлайном, т.е. не теряют
векторную структуру. Также файлы, вставленные через конфиг files включаются непосредственно в бандл сборки. Таким
образом, для отображения картинки в рантайме (в браузере) не требуется делать отдельный сетевой запрос, что увеличивает
скорость загрузки страницы. Данный способ подключения изображений следует использовать для небольших изображений,
например иконок (до 5кб), это позволит ускорить загрузку страниц, но незначительно увеличит размер бандла.

Чтобы получить SVG-разметку нужно:

1) Задекларировать файл в конфиге проекта или движка в формате

```typescript
KEY: PATH
```

где PATH - путь к файлу относительно папки src/svg (например - 'close': 'icons/close.svg').

2) Положить файл по соответствующему пути в src/svg.
3) Вызвать метод FilesService.getSvgByName(KEY). Файлы могут храниться как в проекте, так и в движке. Файл следует
   декларировать там где он лежит. Файлы в проекте имеют больший приоритет, при совпадении имен в проекте и движке будет
   взять файл из проекта.

Метод FilesService.getSvgByName всегда возвращает объект вида:

```typescript
{
    key: string; // переданный ключ
    location:   'local-wlc' | 'local-engine'; // фактическое размещение файла
    htmlString: string; // разметка SVG
}
```

Если файл по переданному ключу не найден, объект будет содержать только поле key.

## Папка root/statiс/images и сервер статики gstatic.

Папка root/statiс/images используется для хранения изображений проекта. Может хранить любые форматы. Файлы из этой папки
не попадают в бандл. Бинарные файлы (jpj, png, gif, webp) загружаются отдельными запросам через атрибут SRC тега IMG.
Файлы SVG, также загружаются отдельным запросом, но могут вставляться в шаблон инлайном, как SVG-разметка.

Чтобы получить файл, нужно вызвать метод FilesService.getFileByUrl(URL), где URL - путь до файла относительно папки
root/statiс/images

Например файл лежит по пути **roots/static/images/icons/sidebar-link-arrow.svg** значит URL должен быть
**icons/sidebar-link-arrow.svg**.

Метод вернет объект вида:

```typescript
{
    key: string; // переданный ключ(url)
    location:   'static' | 'gstatic'; // фактическое размещение файла
    url: string; // относительный путь до изображения
    htmlString: string; // разметка SVG, если запрошенный файл имеет расширение SVG (опционально)
}
```

Файлы могут быть размещены как локально в папке **root/statiс/images**, так и на сервере статики **gstatic**. Метод сам
определит расположение файла и вернет нужный путь. В первую очередь метод проверит **root/statiс/images**.

Важно! В случаях когда метод возвращает путь на сервер **gstatic**, наличие нужного файла не гарантируется и не
проверяется.

## Встраивание изображений в шаблон.

Для непосредственной вставки изображения в шаблон следует использовать компонент **wlc-icon**

Вызывается как атрибут. Принимает параметры:

* **iconUrl** - путь до изображения, аналогичен параметру URL метода **FilesService.getFileByUrl**.
* **iconName** - ключ, указанный в конфиге files, аналогичен параметру **KEY** метода **FilesService.getSvgByName**.

Приоритет у параметра **iconUrl**.

Если целевой файл является бинарным, компонент вставит в шаблон тег IMG с атрибутом **SRC=<_путь до файла_>**.

Если целевой файл SVG, компонент вставит в шаблон разметку SVG.
