# 1.4. Особенности реализации

## WLC Engine - сервис

WLC Engine представляет собой сервис, на основе которого разравачиваются клиентские WLC проекты. В связи с этим его
реализация выполнена таким образом, чтобы была возможность вносить изменения в функционал и внешний вид проекта так,
чтобы не затронуть другие проекты, построенные на WLC Engine.

Кроме того WLC Engine подразумевает постоянное обновление функционала: багфиксинг, добавление новых тем оформления
компонентов, добавление новых модулей и компонентов, расширение функционала существующих модулей и компонентов.

WLC Engine представляет собой конструктор сайта, не имеющий строгой сетки компонентов. Всё задаётся через файлы
конфигурации: положение компонентов на странице, состав каждой страницы, её поведение, внешний вид. При этом "из
коробки" предоставляются готовые файлы конфигурации, полностью настроенные, протестированные и гарантирующие корректную
работу всех частей WLC проекта.

## Стилизация

В основу стилизации (CSS) заложен принцип смены стандартных тем оформления компонентов с внесением в них изменений на
уровне SASS переменных. Т.е. каждый компонент имеет как минимум одну стандартную тему оформления, полностью настроенную
и протестированную отделом QA. Все основные числовые значения (а также другие возможные) вынесены в SASS переменные,
которые можно поменять на уровне проекта, не переписывая CSS компонента. Таким образом сохраняется возможность получения
централизованных обновлений с сохранением кастомного оформления.

Пример: допустим компонент каталога игр имеет стандартное оформление, в котором игры выводятся в 5 столбцов и 6 строк,
имеют скругления блоков 10px, заголовок белого цвета, размер шрифта 20px, расстояние между блоками 20px. Необходимо
сменить эти значения на иные. В классическом варианте можно подключить свой файл стилей CSS в проект и определить
соответствующие стили для данных элементов. Если после рефакторинга кода в новом релизе WLC Engine сменится названия
классов или структура HTML шаблона (как вариант это может понадобиться для фикса багов отображения компонента в браузере
Safari) это может сделать проектные стили недействительными и привести к непредсказуемому результату. В случае изменения
переменных не теряется их привязка к конкретным элементам шаблона и любое обновление произойдёт гладко.

В случае, когда стилизовать компонент одними переменными недостаточно можно пойти классическим путём и определить свои
стили ниже по коду, переопределяя те части CSS кода, что заданы для компонента по умолчанию. Если и этого недостаточно,
есть возможность отключить у компонента все стандартные стили и написать свои.

## Шаблоны компонентов

Каждый компонент имеет свой настроенный HTML шаблон. В случае, если он не подходит для реализации задуманного
функционала, есть возможность назначить компоненту свой шаблон в рамках проекта, не внося изменения в файлы WLC Engine.
Это позволяет получать обновления кодовой базы шаблона в части TS/JS, при этом не зависеть от HTML/CSS кода.
