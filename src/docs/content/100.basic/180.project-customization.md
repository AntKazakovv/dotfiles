# 1.8. Project customization

## Custom components

You can create a new component using **Angular CLI** or manually. A typical component has the following structure:

* [src/custom/components/custom-component/]()
    * [styles/]()
        * [custom-component.component.scss]()
    * [custom-component.component.html]()
    * [custom-component.component.spec.ts]()
    * [custom-component.component.ts]()
    * [custom-component.params.ts]()

To ensure the proper operation of the component, make sure to inherit from an abstract component and then specify `$params/inlineParams/injectParams`.

* `inlineParams` is passed from the following component:
```typescript
<div wlc-custom-component
     [inlineParams]="params"
></div>
```

* `injectParams` are passed either from the configuration file or via **NgComponentOutlet**:
```typescript
...
{
    name: 'custom.wlc-custom-component',
    params: { ... },
},
...
```

* `$params` combine `injectParams` and `inlineParams`. They are merged during `ngOnInit` by calling the `ngOnInit` method of an abstract component and passing `inlineParams` to it. 

The general structure of the initial component is as follows:

**custom-component.component.ts**
```typescript
import {
    Component,
    OnInit,
    Input,
    Inject,
} from '@angular/core';
import {AbstractComponent} from 'wlc-engine/modules/core';
import * as Params from './custom-component.params';
 
@Component({
    selector: '[wlc-custom-component]',
    templateUrl: './custom-component.component.html',
    styleUrls: ['./styles/custom-component.component.scss'],
})
export class CustomComponent extends AbstractComponent implements OnInit {
    @Input() public inlineParams: Params.ICustomCParams;
    public $params: Params.ICustomCParams;
 
    constructor(
        @Inject('injectParams') protected params: Params.ICustomCParams,
    ) {
        super({
            injectParams: params,
            defaultParams: Params.defaultParams,
        });
    }
 
    public async ngOnInit(): Promise<void> {
        super.ngOnInit(this.inlineParams);
    }
}
```

**custom-component.params.ts**
```typescript
import {IComponentParams, CustomType} from 'wlc-engine/modules/core';
export type Theme = 'default' | CustomType;
export type Type = 'default' | CustomType;
export type ThemeMod = 'default' | CustomType;
export interface ICustomCParams extends IComponentParams<Theme, Type, ThemeMod> {}
export const defaultParams: ICustomCParams = {
    class: 'wlc-custom-params',
};
```

Before using the component, make sure to include it in **custom.module.ts:**

**src/custom/custom.module.ts**
```typescript
import {CommonModule} from '@angular/common';
import {NgModule} from '@angular/core';
import {TranslateModule} from '@ngx-translate/core';
import {CoreModule} from 'wlc-engine/modules/core/core.module';
import {CustomComponent} from 'src/custom/components/custom-component/custom-component.component';
export const components = {
    'wlc-cash-request': CustomComponent,
};
 
@NgModule({
    imports: [
        CommonModule,
        TranslateModule,
        CoreModule,
    ],
    providers: [],
    declarations: [
        CustomComponent,
    ],
    exports: [
        CustomComponent,
    ],
})
export class CustomModule {}
```

f you need to modify the ts code of a component, you can inherit from the required component in your custom module and then specify the path to the template and styles in the component's settings.

## Custom states
Pages are generated by **UIRouter** (for more details on this library, please refer to its documentation). The custom configuration file is stored in [src/custom/system/config/custom.states.ts](). A typical configuration file is as follows:

**src/custom/system/config/custom.states.ts**
```typescript
import {StateHelper} from 'wlc-engine/modules/core/system/helpers/state.helper';
export const customStates = {
    'app.custom': {
        url: '/custom',
        resolve: [
            StateHelper.forAuthenticatedResolver(),
        ],
    },
};
```

# 1.8. Кастомизация проекта

## Кастомные компоненты
  Компонент можно создать с помощью **Angular CLI** или создать руками. Примерная структура компонента:

* [src/custom/components/custom-component/]()
    * [styles/]()
        * [custom-component.component.scss]()
    * [custom-component.component.html]()
    * [custom-component.component.spec.ts]()
    * [custom-component.component.ts]()
    * [custom-component.params.ts]()

Для дальнейшей работы компонента необходимо отнаследоваться от  абстрактного компонента и прописать `$params/inlineParams/injectParams`.

* `inlineParams` прокидываются из компонента:
```typescript
<div wlc-custom-component
     [inlineParams]="params"
></div>
```

* `injectParams` попадают из конфига или через NgComponentOutlet:
```typescript
...
{
    name: 'custom.wlc-custom-component',
    params: { ... },
},
...
```

* `$params` - это объединенные `injectParams` и `inlineParams`. Они объединяются на этапе `ngOnInit`, для этого нужно вызвать метод `ngOnInit` абстрактного компонента и передать в него `inlineParams`. 

Ниже представлен скелет стартового компонента:

**[custom-component.component.ts]()**
```typescript
import {
    Component,
    OnInit,
    Input,
    Inject,
} from '@angular/core';
import {AbstractComponent} from 'wlc-engine/modules/core';
import * as Params from './custom-component.params';
 
@Component({
    selector: '[wlc-custom-component]',
    templateUrl: './custom-component.component.html',
    styleUrls: ['./styles/custom-component.component.scss'],
})
export class CustomComponent extends AbstractComponent implements OnInit {
    @Input() public inlineParams: Params.ICustomCParams;
    public $params: Params.ICustomCParams;
 
    constructor(
        @Inject('injectParams') protected params: Params.ICustomCParams,
    ) {
        super({
            injectParams: params,
            defaultParams: Params.defaultParams,
        });
    }
 
    public async ngOnInit(): Promise<void> {
        super.ngOnInit(this.inlineParams);
    }
}
```

**[custom-component.params.ts]()**
```typescript
import {IComponentParams, CustomType} from 'wlc-engine/modules/core';
export type Theme = 'default' | CustomType;
export type Type = 'default' | CustomType;
export type ThemeMod = 'default' | CustomType;
export interface ICustomCParams extends IComponentParams<Theme, Type, ThemeMod> {}
export const defaultParams: ICustomCParams = {
    class: 'wlc-custom-params',
};
```

Перед тем, как использовать компонент, его необходимо подключить в **custom.module.ts:**

**[src/custom/custom.module.ts]()**
```typescript
import {CommonModule} from '@angular/common';
import {NgModule} from '@angular/core';
import {TranslateModule} from '@ngx-translate/core';
import {CoreModule} from 'wlc-engine/modules/core/core.module';
import {CustomComponent} from 'src/custom/components/custom-component/custom-component.component';
export const components = {
    'wlc-cash-request': CustomComponent,
};
 
@NgModule({
    imports: [
        CommonModule,
        TranslateModule,
        CoreModule,
    ],
    providers: [],
    declarations: [
        CustomComponent,
    ],
    exports: [
        CustomComponent,
    ],
})
export class CustomModule {}
```

Если нужно модифицировать ts-код компонента, то можно отнаследоваться от нужного компонента в вашем custom module  и в настройках компонента указать путь до шаблона и стилей.

## Кастомные стейты
За создание страниц отвечает **UIRouter**, подробнее можно посмотреть в документации к самой библиотеке. Файл кастомного конфига располагается по пути: [src/custom/system/config/custom.states.ts](). Пример конгфига представлен ниже: 

**[src/custom/system/config/custom.states.ts]()**
```typescript
import {StateHelper} from 'wlc-engine/modules/core/system/helpers/state.helper';
export const customStates = {
    'app.custom': {
        url: '/custom',
        resolve: [
            StateHelper.forAuthenticatedResolver(),
        ],
    },
};
```
