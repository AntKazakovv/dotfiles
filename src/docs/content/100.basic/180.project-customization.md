# 1.8. Project customization

## Custom components

You can create a new component using **Angular CLI** or manually. A typical component has the following structure:

* [src/custom/components/custom-component/]()
    * [styles/]()
        * [custom-component.component.scss]()
    * [custom-component.component.html]()
    * [custom-component.component.spec.ts]()
    * [custom-component.component.ts]()
    * [custom-component.params.ts]()

To ensure the proper operation of the component, make sure to inherit from an abstract component and then specify `$params/inlineParams/injectParams`.

* `inlineParams` is passed from the following component:
```typescript
<div wlc-custom-component
     [inlineParams]="params"
></div>
```

* `injectParams` are passed either from the configuration file or via **NgComponentOutlet**:
```typescript
...
{
    name: 'custom.wlc-custom-component',
    params: { ... },
},
...
```

* `$params` combine `injectParams` and `inlineParams`. They are merged during `ngOnInit` by calling the `ngOnInit` method of an abstract component and passing `inlineParams` to it.

The general structure of the initial component is as follows:

**custom-component.component.ts**
```typescript
import {
    Component,
    OnInit,
    Input,
    Inject,
} from '@angular/core';
import {AbstractComponent} from 'wlc-engine/modules/core';
import * as Params from './custom-component.params';

@Component({
    selector: '[wlc-custom-component]',
    templateUrl: './custom-component.component.html',
    styleUrls: ['./styles/custom-component.component.scss'],
})
export class CustomComponent extends AbstractComponent implements OnInit {
    @Input() public inlineParams: Params.ICustomCParams;
    public $params: Params.ICustomCParams;

    constructor(
        @Inject('injectParams') protected params: Params.ICustomCParams,
    ) {
        super({
            injectParams: params,
            defaultParams: Params.defaultParams,
        });
    }

    public async ngOnInit(): Promise<void> {
        super.ngOnInit(this.inlineParams);
    }
}
```

**custom-component.params.ts**
```typescript
import {IComponentParams, CustomType} from 'wlc-engine/modules/core';
export type Theme = 'default' | CustomType;
export type Type = 'default' | CustomType;
export type ThemeMod = 'default' | CustomType;
export interface ICustomCParams extends IComponentParams<Theme, Type, ThemeMod> {}
export const defaultParams: ICustomCParams = {
    class: 'wlc-custom-params',
};
```
**[custom-component.component.html]()**
```html
<div>
    <p>Custom component</p>
</div>
```


Before using the component, make sure to include it in **custom.module.ts:**

**src/custom/custom.module.ts**
```typescript
import {CommonModule} from '@angular/common';
import {NgModule} from '@angular/core';
import {TranslateModule} from '@ngx-translate/core';
import {CoreModule} from 'wlc-engine/modules/core/core.module';
import {CustomComponent} from 'src/custom/components/custom-component/custom-component.component';
export const components = {
    'wlc-cash-request': CustomComponent,
};

@NgModule({
    imports: [
        CommonModule,
        TranslateModule,
        CoreModule,
    ],
    providers: [],
    declarations: [
        CustomComponent,
    ],
    exports: [
        CustomComponent,
    ],
})
export class CustomModule {}
```
Consider an example of adding our component through the configuration on the main page of the site:

To display our component on the main page, add it to **layouts.config.ts**

**[config/frontend/02.layouts.config.ts]()**
```typescript
export const $layouts: ILayoutsConfig = {
    'app.home': {
        replaceConfig: true,
        extends: 'app',
        sections: {
            'banner-section': sectionsLib.bannerSection.home,
            'categories': sectionsLib.categories.catalog,
            'content-games-top': sectionsLib.contentGames.homeTop,
            'content-wins': sectionsLib.contentGames.homeWinsTournament,
            'content-games-bottom': sectionsLib.contentGames.homeBottom,
            'total-jackpot': sectionsLib.totalJackpotSection.home,
            //custom component
            'custom-component': {
                container: true,
                components: [
                    {
                        name: 'custom.wlc-custom-component',
                    },
                ],
            },
        },
    },
};
```


If you need to modify the ts code of a component, you can inherit from the required component in your custom module and then specify the path to the template and styles in the component's settings.

## Custom Services

Let's create a custom service for the component that was created above.

The service can also be created using the **Angular CLI** or by hand. Approximate service location:

**[src/custom/services/custom-service.service.ts]()**

For the correct operation of "lazy load" we connect services through **InjectionService**.

Before that, we will connect our service in **custom.module.ts**.

**[src/custom/custom.module.ts]()**

```typescript
import {CustomServiceService} from 'src/custom/services/custom-service.service';
...
export const services = {
    'custom-service-service': CustomServiceService,
};
```

Using a custom service in a component:

```typescript
//import InjectionService and our service
import {InjectionService} from 'wlc-engine/modules/core';
import {CustomServiceService} from 'src/custom/services/custom-service.service';
...
//define them in the constructor
constructor(
    @Inject('injectParams') protected params: Params.ICustomCParams,
    protected injectionService: InjectionService,
    protected customServise: CustomServiceService,
)
...
//connect our service through injectionService
public async ngOnInit(): Promise<void> {
    super.ngOnInit(this.inlineParams);
    this.customServise = await this.injectionService.getService('custom.custom-service-service');
}
```

## Custom states
Pages are generated by **UIRouter** (for more details on this library, please refer to its documentation). The custom configuration file is stored in [src/custom/system/config/custom.states.ts](). A typical configuration file is as follows:

**src/custom/system/config/custom.states.ts**
```typescript
import {StateHelper} from 'wlc-engine/modules/core/system/helpers/state.helper';
export const customStates = {
    'app.custom': {
        url: '/custom',
        resolve: [
            StateHelper.forAuthenticatedResolver(),
        ],
    },
};
```

An example of adding a button with a custom state through the config:

**[config/frontend/02.layouts.config.ts]()**
```typescript
export const $layouts: ILayoutsConfig = {
    'app.home': {
        replaceConfig: true,
        extends: 'app',
        sections: {
            'custom-section': {
                container: true,
                components: [
                    {
                        name: 'core.wlc-button',
                        params: {
                            class: 'custom.btn-class',
                            common: {
                                text: gettext('Go to custom state'),
                                sref: 'app.custom',
                            },
                        },
                    },
                ],
            },
        },
    },
};
```

In html template:

```html
<button wlc-button text="Go to custom state" sref="app.custom">
```

# 1.8. Кастомизация проекта

## Кастомные компоненты
  Компонент можно создать с помощью **Angular CLI** или создать руками. Примерная структура компонента:

* [src/custom/components/custom-component/]()
    * [styles/]()
        * [custom-component.component.scss]()
    * [custom-component.component.html]()
    * [custom-component.component.spec.ts]()
    * [custom-component.component.ts]()
    * [custom-component.params.ts]()

Для дальнейшей работы компонента необходимо отнаследоваться от  абстрактного компонента и прописать `$params/inlineParams/injectParams`.

* `inlineParams` прокидываются из компонента:
```typescript
<div wlc-custom-component
     [inlineParams]="params"
></div>
```

* `injectParams` попадают из конфига или через NgComponentOutlet:
```typescript
...
{
    name: 'custom.wlc-custom-component',
    params: { ... },
},
...
```

* `$params` - это объединенные `injectParams` и `inlineParams`. Они объединяются на этапе `ngOnInit`, для этого нужно вызвать метод `ngOnInit` абстрактного компонента и передать в него `inlineParams`.

Ниже представлен скелет стартового компонента:

**[custom-component.component.ts]()**
```typescript
import {
    Component,
    OnInit,
    Input,
    Inject,
} from '@angular/core';
import {AbstractComponent} from 'wlc-engine/modules/core';
import * as Params from './custom-component.params';

@Component({
    selector: '[wlc-custom-component]',
    templateUrl: './custom-component.component.html',
    styleUrls: ['./styles/custom-component.component.scss'],
})
export class CustomComponent extends AbstractComponent implements OnInit {
    @Input() public inlineParams: Params.ICustomCParams;
    public $params: Params.ICustomCParams;

    constructor(
        @Inject('injectParams') protected params: Params.ICustomCParams,
    ) {
        super({
            injectParams: params,
            defaultParams: Params.defaultParams,
        });
    }

    public async ngOnInit(): Promise<void> {
        super.ngOnInit(this.inlineParams);
    }
}
```

**[custom-component.params.ts]()**
```typescript
import {IComponentParams, CustomType} from 'wlc-engine/modules/core';
export type Theme = 'default' | CustomType;
export type Type = 'default' | CustomType;
export type ThemeMod = 'default' | CustomType;
export interface ICustomCParams extends IComponentParams<Theme, Type, ThemeMod> {}
export const defaultParams: ICustomCParams = {
    class: 'wlc-custom-params',
};
```
**[custom-component.component.html]()**
```html
<div>
    <p>Custom component</p>
</div>
```

Перед тем, как использовать компонент, его необходимо подключить в **custom.module.ts:**

**[src/custom/custom.module.ts]()**
```typescript
import {CommonModule} from '@angular/common';
import {NgModule} from '@angular/core';
import {TranslateModule} from '@ngx-translate/core';
import {CoreModule} from 'wlc-engine/modules/core/core.module';
import {CustomComponent} from 'src/custom/components/custom-component/custom-component.component';
export const components = {
    'wlc-cash-request': CustomComponent,
};

@NgModule({
    imports: [
        CommonModule,
        TranslateModule,
        CoreModule,
    ],
    providers: [],
    declarations: [
        CustomComponent,
    ],
    exports: [
        CustomComponent,
    ],
})
export class CustomModule {}
```

Рассмотрим пример добавления нашего компонента через конфиги на главной странице сайта:

Чтобы отобразить наш компонент на главной странице добавим его в **layouts.config.ts**

**[config/frontend/02.layouts.config.ts]()**
```typescript
export const $layouts: ILayoutsConfig = {

            'content-games-bottom': sectionsLib.contentGames.homeBottom,
```

Если нужно модифицировать ts-код компонента, то можно отнаследоваться от нужного компонента в вашем custom module  и в настройках компонента указать путь до шаблона и стилей.
## Кастомные сервисы

Создадим кастомный сервис для компонента, который создавался выше.

Сервис также можно создать с помощью **Angular CLI** или руками. Примерное расположение сервиса:

**[src/custom/services/custom-service.service.ts]()**

Для корректной работы "ленивой загрузки (lazy load)" подключаем сервисы через **InjectionService**.

Перед этим подключим наш сервис в **custom.module.ts**.

**[src/custom/custom.module.ts]()**

```typescript
import {CustomServiceService} from 'src/custom/services/custom-service.service';
...
export const services = {
    'custom-service-service': CustomServiceService,
};
```

Использование кастомного сервиса в компоненте:

```typescript
//импортируем InjectionService и наш сервис
import {InjectionService} from 'wlc-engine/modules/core';
import {CustomServiceService} from 'src/custom/services/custom-service.service';
...
//определяем их в конструкторе
constructor(
    @Inject('injectParams') protected params: Params.ICustomCParams,
    protected injectionService: InjectionService,
    protected customServise: CustomServiceService,
)
...
//подключаем наш сервис через injectionService
public async ngOnInit(): Promise<void> {
    super.ngOnInit(this.inlineParams);
    this.customServise = await this.injectionService.getService('custom.custom-service-service');
}
```
