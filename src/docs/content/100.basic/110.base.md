# 1.1. Basic principles and definitions

## Description

WLC-Engine is an engine for the quick deployment of ready-to-use WLC applications. Its design allows you to create even the most sophisticated projects using only configuration files. At the same time, you will be able to update your projects from the unified dashboard. This engine serves as the library and as local files for your projects thanks to its flexible project assembly principles. For more details, please refer to "Project Assembly".

## Principles

* No page templates. The page canvas is generated automatically based on the configuration.
* States are created dynamically in accordance with the project configuration and the data retrieved from the server.
* A bunch of preset components and configuration files that describe the page structure is used for composing pages.
* All components offer 4 configuration levels: "Default", "Engine-wide", "Project-wide", and "Custom". Each subsequent level supersedes the previous one.
* All components feature the preset styles and ready-to-use templates.
* The templates and styles for any component are customizable and backward compatible with the default ones, so that you could modify and update them confidently. However, default functionalities are not customizable. To make changes to the functionality, you should use the TS inheritance mechanism to write a custom component inherited from the default one within your project.
* Basic services, core settings and other elements directly related to the engine itself cannot be modified within projects. To modify or extend the existing functionality, you should use only the TS inheritance mechanism, so that all changes you make could not affect the default out-of-the-box service.

In general, projects on the WLC-Engine are assembled from the ready-made components. The page layout (state) as well as the visual appearance (style) and behavior of each component can be set in the project configuration to meet your customer's demands and expectations with zero coding.

## Styles

By default, all components have the consistent look-and-feel depending on the selected theme, but you are free to adjust the visual appearance of a particular component (e.g. change the color of a given button) or even completely redefine the CSS styles at the component level. This can be done both for a given instance of a component (e.g. only the sign-in buttons in the header) and for all components of a given type (e.g. all buttons on the site).

## Templates

A component template can include several look-and-feel options and switch between them depending on the input parameter (type). For more details, please refer to [Components](../5.-components.html)

# 1.1. Основные принципы и понятия

## Описание

WLC-Engine - движок, на основе которого возможно быстро разворачивать готовые WLC приложения. Он создан таким образом,
чтобы можно было, используя только файлы конфигурации, создавать проекты практически любой сложности. При этом
сохраняется возможность централизованного обновления проекта. Т.е. с одной стороны движок — это библиотека, с другой
стороны сборка проекта сделана таким образом, что файлы движка используются как локальные файлы для проекта. Более
подробно смотри "Сборка проекта".

## Принципы

* Нет шаблона страниц. Сетка страниц создаётся автоматически на основе конфигурации.
* Стейты формируются динамически, на основе конфигурации проекта и получаемых данных от сервера.
* Для формирования страниц используется связка преднастроенных компонентов и конфигурационных файлов, описывающих
  структуру.
* Все компоненты имеют 4 степени конфигурирования: "по умолчанию", "глобально для движка", "глобально для проекта", "
  кастомно для проекта". Каждый последующий уровень имеет приоритет перед предыдущим.
* Все компоненты имеют преднастроенные стили оформления и подготовленные шаблоны.
* Все компоненты имеют возможность кастомизации шаблонов и стилей с сохранением обратной совместимости со стандартным
  состоянием, что позволяет получать обновления функционала. Функционал кастомизировать нельзя. Для внесения изменений
  возможно написать свой компонент в рамках проекта и используя систему наследования TS получать обновления функционала.
* Базовые сервисы, настройки и другие элементы, явно влияющие на работу движка, не подлежат изменению в рамках проекта.
  Внесение дополнительного функционала или изменение существующего возможно только через систему наследования TS, при
  этом точечно меняется/дополняется функционал, сохраняя работу всего сервиса "из коробки".

Таким образом проекты на WLC-Engine представляют собой конструктор, собираемый из готовых компонентов. Расположение
компонентов на странице (стейте), внешний вид компонента (используемая тема оформления), поведение в рамках конфигурации
компонента — задаётся на уровне конфигурации проекта и позволяет настраивать под пожелания клиента существующие элементы
без внесения изменений в код.

## Стилизация

Стилевое оформление всех компонентов по умолчанию завязано на единую дизайн-систему, при этом подразумевается точечное
изменение как переменных стилей конкретного компонента (например изменение цвета конкретной кнопки), так и полное
переопределение стилей CSS на уровне компонента. Это возможно как на уровне конкретного экземпляра компонента (например
только кнопки логина в шапке), так и всех компонентов данного типа (все кнопки на сайте).

## Шаблоны

Шаблон компонента может включать в себя несколько вариантов оформления и отображение того или иного варианта зависит от
переданного в компонент параметра **type**. (подробнее в разделе [Компоненты](../5.-components.html))
