# 1.9. Standard banners and workflows

## How to set up buttons in banners

If you want to update a button with a link, specify the following parameters in the template, where `sref` is the link to the state:
```html
<button wlc-button text="Deposit" sref='app.profile.cash.deposit'> 
```
When you need to display a given modal window, use the following parameters:
```html
<button wlc-button text="Sign up" *auth="false" [event]="{name: 'SHOW_MODAL', data: 'signup'}"></button>
```
where `data` contains the modal window name.   
`auth `displays or hides the modal window for the authorized user.
## 1. Banner - 1:
```html
<div class="banner banner-1" wlc-parallax>
    <div class="banner_bg">
        <img src="/gstatic/wlc/banners/default/1920_tinified/banner-1/banner-1.jpg">

        <div class="banner__description">€/$2,000 Welcome Bonus</div>
        <button wlc-button text="Sign up" *auth="false" 
        [event]="{name: 'SHOW_MODAL', data: 'signup'}"></button>
    <button wlc-button text="Deposit" *auth="true" sref='app.profile.cash.deposit'></button>
    </div>
``` 

# 1.9. Стандартные баннеры и описание работы.

## Как настраивать кнопки в баннерах.

Если Вам нужно, чтобы кнопка вела на какую-нибудь ссылку, то нужно в шаблоне прописать, где `sref` - ссылка на стейт:
```html
<button wlc-button text="Deposit" sref='app.profile.cash.deposit'> 
```
Если нужно вызвать определенную модалку, то:
```html
<button wlc-button text="Sign up" *auth="false" [event]="{name: 'SHOW_MODAL', data: 'signup'}"></button>
```
Где в `data` передаем название модального окна.  
 `auth` - отвечает за то, показывать ли для авторизированного пользователя или нет.
