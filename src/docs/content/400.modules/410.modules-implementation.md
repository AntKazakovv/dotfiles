# 4.1. Implementation features

## Module structure

* **![](../assets/folder.svg) components** - _includes the module's components_
* **![](../assets/folder.svg) constants** - _constants_
* **![](../assets/folder.svg) directives** - _directives_
* **![](../assets/folder.svg) pipes** - _pipes/filters_
* **![](../assets/folder.svg) system** - _includes files that directly affect the operation of the module or the entire project. Any changes to these files may be dangerous for the entire project._
    * **![](../assets/folder.svg) animations** - _animations_
    * **![](../assets/folder.svg) classes** - _includes separate classes combining related features within the module (optional directory)_
    * **![](../assets/folder.svg) config** - _module's configuration_
    * **![](../assets/folder.svg) helpers** - _helpers_
    * **![](../assets/folder.svg) interfaces** - _includes the module's common data interfaces_
    * **![](../assets/folder.svg) models** - _data models_
    * **![](../assets/folder.svg) services** - _module's services_
* name.module.ts - _the module's file_
* index.ts - _is the indexing file_

## Description

All directories may be optional depending on a given implementation and purpose of your module.  

"System" is a special directory, which contains files that are crucial for the operation of the entire project. Unlike files in other directories, making invalid changes to the entities stored in "system" will cause malfunctioning of the whole project rather than an individual component.  

You can replace HTML and SASS files with your local project files without making changes to WLC Engine. This allows you to customize any project for each client without losing backward compatibility or making changes the business logic.  

If you need to make changes to the business logic, i.e. to the TS files, consider using the **extends** feature to inherit your classes from the system ones. Making changes directly to the TS files of WLC Engine is prohibited.

# 4.1. Особенности реализации

## Структура модуля

* **![](../assets/folder.svg) components** - _компоненты модуля_
* **![](../assets/folder.svg) constants** - _константы_
* **![](../assets/folder.svg) directives** - _директивы_
* **![](../assets/folder.svg) pipes** - _пайпы/фильтры_
* **![](../assets/folder.svg) system** - _файлы, напрямую влияющие на работу модуля или всего проекта. Любое изменение в
  данных файлах потенциально опасно для всего проекта._
    * **![](../assets/folder.svg) animations** - _анимации_
    * **![](../assets/folder.svg) classes** - _отдельные классы, объединяющие некий сводный функционал в рамках модуля (
      необязательная директория)_
    * **![](../assets/folder.svg) config** - _конфигурация модуля_
    * **![](../assets/folder.svg) helpers** - _хелперы_
    * **![](../assets/folder.svg) interfaces** - _интерфейсы данных, общих для модуля_
    * **![](../assets/folder.svg) models** - _модели данных_
    * **![](../assets/folder.svg) services** - _сервисы модуля_
* name.module.ts - _файл модуля_
* index.ts - _индексный файл_

## Описание

Все директории могут быть необязательными. Это зависит от конкретной реализации и назначения модуля.

System - специальная директория, файлы в которой являются критичными для работы всего проекта в целом. Если остальные
можно менять и в большинстве случаев ошибки в них приведут к некорректной работе конкретного компонента, то все сущности
в system с большей вероятностью повлияют глобально.

Для файлов HTML и SASS имеется возможность подмены на локальные файлы проекта, т.е. их можно заменить на свои, не внося
изменения в WLC Engine. Это позволяет настраивать любой проект под клиента, не ломая обратной совместимости и не изменяя
логику.

Если необходимо внести изменение в логику, т.е. в TS файлы, то используется возможжность наследования классов **
extends**. Прямой возможности изменения TS файлов WLC Engine не предусмотрено.
