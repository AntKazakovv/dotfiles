# 5.2. Параметры компонента

## Пример конфигурации компонента

```typescript
import {IComponentParams, CustomType} from 'wlc-engine/classes/abstract.component';

export type Type = 'default' | 'resolved' | 'rejected' | 'pending' | 'disabled' | CustomType;
export type Theme = 'default' | 'skew' | 'rounding' | 'circled' | CustomType;
export type Size = 'default' | 'small' | 'big' | CustomType;
export type ThemeMod = 'default' | 'secondary' | CustomType;
export type Index = number | string | null;
export type AutoModifiers = Theme | Size | ThemeMod | 'loading';
export type CustomMod = string;
export type Modifiers = AutoModifiers | CustomMod | null;

export interface IButtonCParams extends IComponentParams<Theme, Type, ThemeMod> {
    modifiers?: Modifiers[];
    common?: {
        themeMod?: ThemeMod;
        size?: Size;
        icon?: string;
        index?: Index;
        text?: string;
        customModifiers?: CustomMod;
    };
}

export const defaultParams: IButtonCParams = {
    moduleName: 'base',
    componentName: 'wlc-button',
    class: 'wlc-btn',
    common: {
        size: 'default',
    },
};
```

### Описание типов:

* **IComponentParams** - стандартный интерфейс базовых параметров, относящихся ко всем компонентам.
* **CustomType** - тип компонента с полностью сброшенными стилями и другими параметрами.
* **type Type** - перечисление возможных типов компонента. Задаётся жёстко в коде и для каждого типа, при необходимости,
  создаются свои SCSS стили.
* **type Theme** - перечисление возможных тем компонента. Задаётся жёстко в коде и для ой темы разрабатываются свои SCSS
  стили.
* **type ThemeMod** - модификатор темы. Может быть только один из перечисленных в данном типе. Т.е. на данный момент не
  предполагается несколько модификатором одной темы. Если возникает такая необходимость - создаётся новая тема.
* **Остальные типы** - особенности реализации конкретного компонента.

### Описание интерфейса параметров компонента:

* Интерфейс обязательно должен наследоваться от **IComponentParams**.
* Блок common создаётся для параметров, относящихся со всему компоненту в целом. Необязательный параметр.

### Описание стандартных параметров компонента:

* **moduleName**, **componentName** - имя модуля и имя компонента. Используется если компонент имеет глобальные
  параметры (передаваемые из конфигурации проекта в каждый компонент). Необязательный параметр.
* **class** - CSS класс компонента. Передаётся автоматически в параметр компонента **this.$class** в абстрактном
  компоненте. Обязательный параметр.
