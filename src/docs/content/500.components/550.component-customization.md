# 5.5. Customizing component templates and styles

The functionality allows you to include custom templates and styles in the existing components.  

Each time you build the project (npm run dev), the system creates the custom folder in the project structure, where the structure of modules from the engine is duplicated in the following way:

1. Each file gets the "~" prefix to its name.
2. Note that only components are added to "custom". Services, configurations, etc. are not customizable.
3. Each *.ts file gets the "~readonly_" prefix. Files of this type are not customizable, and the builder ignores any changes made to them. They are required for the proper operation of syntax highlighting in templates in the IDE.
4. All files stored in the "custom" folder and having the "~" prefix are ignored by the version control system.
5. If the component has subfolders, then the folder structure is copied.

## Customizing component templates
To edit the template, remove the prefix from the name of the corresponding file (i.e. the **`~`** character at the beginning of the file name).

## Customizing component styles
To edit the styles, remove the prefix from the name of the corresponding file (i.e. the **`~`** character at the beginning of the file name).  

To apply the styles of the component from the engine, import the styles from the engine into your custom file.  

Once you've included any custom files, make sure to rebuild the project!

# 5.5. Кастомизация шаблонов и стилей компонента

Функционал позволяет подключать кастомные шаблоны и стили к существующим компонентам.

Во время каждой сборки (npm run dev), в структуре проекта создается папка **custom**, в которой повторяется структура
модулей из движка, со следующими особенностями:

1. К каждому файлу добавляется префикс "**~**".
2. В "**custom**" попадают только компоненты. Сервисы, конфиги и т.п. не кастомизируются.
3. К файлам **\*.ts** добавляется префикс "**~readonly_**". Файлы этого типа не кастомизируются, их изменения не
   учитываются сборщиком. Они нужны для нормальной работы подсветки синтаксиса в шаблонах в IDE.
4. Все файлы, расположенные в папке "**custom**", с префиксом "**~**" игнорируются системой контроля версий.
5. Если компонент имеет вложенные папки, структура папок повторяется.

## Кастомизация шаблона компонента
Для изменения шаблона необходимо убрать префикс у соответствующего файла (в начале имени символ **~**).

## Кастомизация стилей компонента
Для изменения стилей необходимо убрать префикс у соответствующего файла (в начале имени символ **~**).

Для применения стилей компонента из движка, в кастомный файл импортировать стили из движка.

После подключения кастомных файлов необходимо перезапустить сборку!
