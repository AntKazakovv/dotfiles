# 5.5. Кастомизация шаблонов и стилей компонента

Функционал позволяет подключать кастомные шаблоны и стили к существующим компонентам.

Во время каждой сборки (npm run dev), в структуре проекта создается папка **custom**, в которой повторяется структура
модулей из движка, со следующими особенностями:

1. К каждому файлу добавляется префикс "**~**".
2. В "**custom**" попадают только компоненты. Сервисы, конфиги и т.п. не кастомизируются.
3. К файлам **\*.ts** добавляется префикс "**~readonly_**". Файлы этого типа не кастомизируются, их изменения не
   учитываются сборщиком. Они нужны для нормальной работы подсветки синтаксиса в шаблонах в IDE.
4. Все файлы, расположенные в папке "**custom**", с префиксом "**~**" игнорируются системой контроля версий.
5. Если компонент имеет вложенные папки, структура папок повторяется.

## Кастомизация шаблона компонента
Для изменения шаблона необходимо убрать префикс у соответствующего файла (в начале имени символ **~**).

## Кастомизация стилей компонента
Для изменения стилей необходимо убрать префикс у соответствующего файла (в начале имени символ **~**).

Для применения стилей компонента из движка, в кастомный файл импортировать стили из движка.

После подключения кастомных файлов необходимо перезапустить сборку!
