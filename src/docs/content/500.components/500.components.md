# 5. Components

## Description

The WLC Engine components are the key building blocks for your projects. Thanks to the configuration features, you can build your website's template by placing components on pages in any order. The placement order is determined only by the configuration and does not depend on the HTML code. This is the key difference between building an application in the Angular framework and in WLC Engine.

Thanks to this approach, changes to page templates can be made individually for each project, while keeping backward compatibility even after major updates to WLC Engine. It also allows you to make target changes to specific pages without affecting other sections.

The structure of each page is also defined by the configuration, so that you could create even the most sophisticated templates. In a nutshell, you can use the configuration to define and set up sections of your website and their contents (since they consist of components) and then use the CSS styles to arrange these sections and components in any order you want.

## Implementation features

* You can add a component to your website either via the configuration (by binding the component to a particular state) or inline (if the component supports this feature). By default, all components support adding via the configuration, but you can always make them support adding via the HTML code. For example, the wlc_button component, which is usually part of other components, is added directly to the HTML template, but you can also add a button directly through the website configuration, e.g. to the footer.
* Each your component must inherit from the corresponding abstract component that implements additional functionality.
* Each component features at least one preset and validated default theme.

## Key terms and definitions

Name                    | System name      | Description
--------------------------- | ----------------- | -------------
Abstract component       | AbstractComponent | The abstract component. It is used in the most engine components. This component is designed to work with the parameters of other components and with modifiers.
Base class               | $class            | It is the main CSS class of a component. This class is installed on the HOST object of the component via the abstract class.абстрактный класс.
Modifiers               | Modifiers         | These are modifiers that define the main functionality or appearance of a component. All modifiers are displayed as the corresponding CSS classes, but not all of them affect the visual appearance of the component in the HTML code. Some of the modifiers are responsible for the functional differences in the component's behavior while also allowing for its style customization whenever necessary.
Theme                        | theme             | This modifier is responsible for styling a component in accordance with the approved look-and-feel. Themes may differ significantly both in style and HTML code (whenever necessary). This can be achieved by using different HTML templates.
Theme modifier           | themeMod          | This is a style modifier for making changes to the current theme while keeping the default (theme-specific) behavior.
Type                        | type              | It sets the functional type of the component. Types significantly differ in operation for the component, e.g. "on hover" vs. "on click". Types are also displayed in CSS modifiers.
Customizable modifier   | customMod         | Any modifier required to customize the component's style.
Standard parameters       | defaultParams     | The parameters which are set by default directly in the component's program code.
Injectable parameters     | injectParams      | Parameters which are passed to the component via the global LayoutService configuration.
Inline parameters            | Инлайн параметры  | Parameters passed to the component via HTML templates using the @Input() decorator.

# 5. Компоненты

## Описание

Компонент WLC Engine - основная строительная единица. Благодаря системе конфигурирования шаблон сайта строится путём
размещения компонентов на странице в произвольном порядке. Порядок определяется только конфигурацией и никак не связан с
HTML кодом. Это отличает классическое построение приложения на фреймворке Angular от WLC Engine.

Благодаря такому подходу изменения в шаблон страницы вносится индивидуально для каждого проекта с сохранением обратной
совместимости при обновлении WLC Engine. Так же это позволяет точечно вносить изменения на страницы, не затрагивая
другие разделы.

Сама структура страницы - также задаётся конфигурацией, что позволяет создавать шаблоны практически любой сложности.
Если говорить простым языком, то через конфигурацию задаются секции сайта и их состав (они состоят из компонентов), а
стилями CSS можно расположить эти секции и компоненты в любом порядке.

## Особенности

* Компонент может вставляться на сайт как через конфигурацию (привязка к стейту), так и инлайном (если компонент написан
  для этой возможности). По умолчанию все компоненты создаются для вставки через конфигурацию, но всегда могут быть
  дополнены функционалом вставки через HTML код. Примеом является компонент _wlc_button_, который в большинстве случаев
  является частью других клмпонентов и вставляются непосредственно в HTML шаблоне, но так же сохраняется возможность
  вставки кнопки прямо через конфигурацию сайта, например в футере.
* Компонент должен наслодоваться от абстрактного компонента, реализующий дополнительный функционал.
* Компонент имеет как минимум одну предустановленную настроенную и проверенную тему оформления.

## Основные понятия

Название                    | Наименование      | Описание
--------------------------- | ----------------- | -------------
Абстрактный компонент       | AbstractComponent | Абстрактный компонент. Используется в большинстве компонентов движка. Предназначен для работы с параметрами компонентов, а так же с системой модификаторов.
Базовый класс               | $class            | Основной CSS класс компонента. Устанавливается на HOST объекте компонента через абстрактный класс.
Модификаторы                | Modifiers         | Модификаторы, определяющие основной функционал или оформление компонента. Все модификаторы отображаются в виде соответствующих CSS классов, при этом не все участвуют в визуальном оформлении HTML кода. Часть модификаторов отвечают за функциональные различия поведения компонента, сохраняя при этом возможность индивидуального стилевого оформления, если такая необходимость возникнет.
Тема                        | theme             | Модификатор, отвечающий за оформление компонента согласно согласованного дизайна. Темы имеют между собой значительные различия как в стилях, так и в HTML коде (при необходимости). Это достигается использованием разных HTML шаблонов.
Модификатор темы            | themeMod          | Модификатор стилей, используемый для внесения изменений в рамках одной темы, сохраняя при этом стандартное (в рамках темы) поведение.
Тип                         | type              | Устанавливает функциональный тип компонента. Типы имеют значительное отличие в работе, например работа компонента "по наведению" и "по клику". Типы так же отображаются в CSS модификаторах.
Настраиваемый модификатор   | customMod         | Любой необходимый для индивидуальной стилизации компонента модификатор.
Стандартные параметры       | defaultParams     | Параметры, устанавливаемые по умолчанию, непосредственно в коде компонента.
Инжектируемые параметры     | injectParams      | Параметры, передаваемые компоненту через глобальную конфигурацию LayoutService.
Инлайн параметры            | Инлайн параметры  | Параметры, передаваемые в компонент через HTML шаблоны, используя @Input() декоратор.
