# 5.8.0. Form wrapper component

**Form - wrapper компонент** - это компонент на основе **layout** - компонента, через параметры в конфиге он принимает непосредственно компоненты формы.  
Компонент не имеет дефолтных параметров, через параметр принимает конфиг, который компонент должен сформировать. 

  Пример компонента, используемого form-wrapper компонент. [Feedback-form](https://vcsxa.egamings.com/wlc/wlc-engine/-/tree/master/src/modules/core/components/feedback-form)

## Параметры компонента:

* `config` - Конфиг, описывающий конкретную форму.
```typescript
export const profileForm: IFormWrapperCParams = {
    class: 'wlc-form-wrapper',
    components: [
        {
            name: 'core.wlc-input',
            params: {
                common: {
                    placeholder: 'Email',
                },
                locked: true,
                name: 'email',
                validators: ['required', 'email'],
                exampleValue: 'dasha.kot@egamings.com',
            },
        },
        {
            name: 'core.wlc-button',
            params: {
                name: 'submit',
                common: {
                    text: 'Save changes',
                },
            },
        },
    ],
};
```

* `ngSubmit` - Функция, которая будет вызвана при отправке формы. 
```typescript
public async ngSubmit(form: FormGroup): Promise<boolean> {
    alert('cool');
}
```

* `formData` - Получаемые данные, если нужно заполнить форму уже сформированными данными.
```typescript
[formData]="userProfile"
```

### В качестве примера предоставлена форма профиля:
```typescript
<div wlc-form-wrapper [config]="config" [formData]="userProfile" [ngSubmit]="ngSubmit.bind(this)"></div>
```

## Компонент принимает три параметра:

1. `Config `- набор компонентов и валидаторов. Валидаторы могут быть как глобальными, так и для конкретного поля. Например:
```typescript
{
    class: 'wlc-form-wrapper',
    components: [
            {
                 name: 'core.wlc-input',
                 params: {
                     theme: 'placeholder-shown',
                     common: {
                         placeholder: 'Your name *',
                     },
                     name: 'senderName',
                     validators: ['required'],
                     exampleValue: 'Example: Ivan',
                 },
            },
            {
                 name: 'core.wlc-input',
                 params: {
                     theme: 'placeholder-shown',
                     common: {
                         placeholder: 'Password',
                         type: 'password',
                     },
                     name: 'currentPassword',
                     validators: ['required'],
                 },
            },
            {
                 name: 'core.wlc-input',
                 params: {
                     theme: 'placeholder-shown',
                     common: {
                         placeholder: 'Confirm password',
                         type: 'password',
                     },
                     name: 'confirmPassword',
                     validators: ['required'],
                 },
            },
    ],
    validators: [
        {
               name: 'matchingFields',
               options: ['currentPassword', 'confirmPassword'],
        }
    ]
};
```

2. `ngSubmit` - метод, который будет вызываться по нажатию на кнопку подтверждения. В качестве входящих параметров принимает экземпляр формы. 

3. `formData` - набор полей для авто заполнения формы. 


## Разберем конкретнее конфиг формы:

  В поле `components` перечислены элементы формы, они полностью повторяет **layout - config.**

### **Элемент input:**
```typescript
{
    name: 'core.wlc-input',
    params: {
        common: {
            placeholder: 'Last name',
        },
        name: 'lastName',
        validators: ['required'],
        exampleValue: 'Ivanov',
    },
},
```
`name` - Название компонента;  
`common: { placeholder: 'Last name', },` - Плейсхолдер элемента;  
`params.name` - Имя поля формы, обязательный параметр;  
`validators` - Массив валидаторов, применимых к данному полю. Можно указать просто перечисление валидаторов, либо передать объект с именем и параметром валидатора. Например:
```typescript
{
    name: 'core.wlc-input',
    params: {
        common: {
            placeholder: 'First name',
        },
        name: 'firstName',
        validators: ['required',
            {
                name: 'minLength',
                options: 2,
            },
            {
                name: 'regExp',
                options: new RegExp('^[A-Za-z]$'),
            },
        ],
        exampleValue: 'Ivan',
    },
},
```
`exampleValue` - Значение, которое будет выводится перед тем, как юзер ввел данные, так называемый пример;

### **Элемент select:**
```typescript
{
    name: 'core.wlc-select',
    params: {
        common: {
            placeholder: 'Gender',
        },
        locked: true,
        name: 'gender',
        validators: ['required'],
        options: 'genders',
    },
},
```

Поле                | Описание
--------------------|---------------------
`name`	            | Название компонента;
`common.placeholder`| Плейсхолдер селекта;
`locked`	        | Запрещает при имеющемся значении изменить данные;
`params.name`       |	Название поля в форме, поле является обязательным;
`validators`        |	Смотри элемент input;
`options`           | Заранее заготовленные константы с значениями, объявленные в select - component, на данный момент доступны - ( `currencies`, `countries`, `genders` ) 
  
### **Глобальные валидаторы:**
  Можно установить валидацию на все поля формы, которые будут применяться ко всем. Для этого нужно на уровне конфига формы задать ключ **validators** и указать нужные валидаторы. Пример сравнения двух полей через глобальные валидаторы:

```typescript
validators: [
        {
               name: 'matchingFields',
               options: ['currentPassword', 'confirmPassword'],
        }
    ]
```

В поле **options** нужно вставить имена элементов, которые нужно сравнить.

### Валидаторы:
  Список доступных валидаторов на данных момент. Актуальный список в **validation.service.ts** (core module):

Название            | Описание
--------------------|------------------------------
`emailUnique`       | Проверка уникальности email;
`loginUnique`       | Проверка уникальности login;
`password `         | Соответствие пароля сложности;
`required`          | Обязательность поля;
`minLength `        | Минимальная длина поля;
`maxLength`         | Максимальная длина поля;
`regExp`            | Соответствие значения выбранному паттерну;
`matchingFields`    | Совпадение значения двух полей;
`email`             | Проверка на корректность написания email;
