# 5.8. Form wrapper component

**Form-wrapper component** is the component which is based on the layout component and accepts the form components via the configuration parameters.  
Since this component doesn't have default parameters, it accepts the configuration, which the component must generate, as parameter.   
Example of a component used by form-wrapper. [Feedback-form](https://vcsxa.egamings.com/wlc/wlc-engine/-/tree/master/src/modules/core/components/feedback-form)

## Component parameters:

* `config` - is the configuration that describes a given form.
```typescript
export const profileForm: IFormWrapperCParams = {
    class: 'wlc-form-wrapper',
    components: [
        {
            name: 'core.wlc-input',
            params: {
                common: {
                    placeholder: 'Email',
                },
                locked: true,
                name: 'email',
                validators: ['required', 'email'],
                exampleValue: 'dasha.kot@egamings.com',
            },
        },
        {
            name: 'core.wlc-button',
            params: {
                name: 'submit',
                common: {
                    text: 'Save changes',
                },
            },
        },
    ],
};
```

* `ngSubmit` - is the function to be called when the form is being submitted. 
```typescript
public async ngSubmit(form: FormGroup): Promise<boolean> {
    alert('cool');
}
```

* `formData` - includes the input data being obtained when the form must be filled in with the previously generated data.
```typescript
[formData]="userProfile"
```

### This is an example of a profile form:
```typescript
<div wlc-form-wrapper [config]="config" [formData]="userProfile" [ngSubmit]="ngSubmit.bind(this)"></div>
```

## The component accepts three input parameters:

1. `Config `- is a set of components and validators. Validators can be either global or field-specific. For example:
```typescript
{
    class: 'wlc-form-wrapper',
    components: [
            {
                 name: 'core.wlc-input',
                 params: {
                     theme: 'placeholder-shown',
                     common: {
                         placeholder: 'Your name *',
                     },
                     name: 'senderName',
                     validators: ['required'],
                     exampleValue: 'Example: Ivan',
                 },
            },
            {
                 name: 'core.wlc-input',
                 params: {
                     theme: 'placeholder-shown',
                     common: {
                         placeholder: 'Password',
                         type: 'password',
                     },
                     name: 'currentPassword',
                     validators: ['required'],
                 },
            },
            {
                 name: 'core.wlc-input',
                 params: {
                     theme: 'placeholder-shown',
                     common: {
                         placeholder: 'Confirm password',
                         type: 'password',
                     },
                     name: 'confirmPassword',
                     validators: ['required'],
                 },
            },
    ],
    validators: [
        {
               name: 'matchingFields',
               options: ['currentPassword', 'confirmPassword'],
        }
    ]
};
```

2. `ngSubmit` - is the method that will be called upon clicking on the confirmation button. It accepts a form instance as input parameters.

3. `formData` - is a set of fields to be auto-populated in the form. 


## Let's analyze the form configuration in more detail:

The `components` field lists the form elements exactly as in layout-config.

### The "input" element:
```typescript
{
    name: 'core.wlc-input',
    params: {
        common: {
            placeholder: 'Last name',
        },
        name: 'lastName',
        validators: ['required'],
        exampleValue: 'Ivanov',
    },
},
```
`name` - is the component's name; 
`common: { placeholder: 'Last name', },` - is the element's placeholder;  
`params.name` - is the form's field name (mandatory)  
`validators` - is the array of validators to be applied to a given field. You can either simply specify an enumeration of validators, or pass an object containing the validator's name and parameter. Example:
```typescript
{
    name: 'core.wlc-input',
    params: {
        common: {
            placeholder: 'First name',
        },
        name: 'firstName',
        validators: ['required',
            {
                name: 'minLength',
                options: 2,
            },
            {
                name: 'regExp',
                options: new RegExp('^[A-Za-z]$'),
            },
        ],
        exampleValue: 'Ivan',
    },
},
```
`exampleValue` - is the example to be displayed to the user before he/she enters the data;

### The "select" element:
```typescript
{
    name: 'core.wlc-select',
    params: {
        common: {
            placeholder: 'Gender',
        },
        locked: true,
        name: 'gender',
        validators: ['required'],
        options: 'genders',
    },
},
```

Поле                | Описание
--------------------|---------------------
`name`	            | Component's name;
`common.placeholder`| Selector's placeholder;
`locked`	        | When filled in, prohibits any changes to the data;
`params.name`       | Field name on the form **(mandatory)**
`validators`        | See the "`input`" element;
`options`           | The preset constants with values defined in select-components; currently, the following values are supported: (`currencies`, `countries`, `genders`) 
  
### Global validators:
You can set up validation for all form fields. To do so, specify the validators key at the form configuration level and then indicate the validators you need. This is an example of how to compare two fields using global validators:

```typescript
validators: [
        {
               name: 'matchingFields',
               options: ['currentPassword', 'confirmPassword'],
        }
    ]
```

Add the names of elements to be compared to the `options` field.

### Validators:
This is a list of currently supported validators: The up-to-date list in **validation.service.ts** (core module):

Название            | Описание
--------------------|------------------------------
`emailUnique`       | Checks whether the email is unique;
`loginUnique`       | Checks whether the login is unique;
`password `         | Checks whether the password is strong enough;
`required`          | Denotes a mandatory field;
`minLength `        | The minimum length of the field;
`maxLength`         | The maximum length of the field;
`regExp`            | Checks for a value that matches the selected pattern;
`matchingFields`    | Checks for fields having the same values;
`email`             | Checking the email address for valid spelling;

Название              | Описание
----------------------|------------------------------

# 5.8. Form wrapper component

**Form - wrapper компонент** - это компонент на основе **layout** - компонента, через параметры в конфиге он принимает непосредственно компоненты формы.  
Компонент не имеет дефолтных параметров, через параметр принимает конфиг, который компонент должен сформировать. 

  Пример компонента, используемого form-wrapper компонент. [Feedback-form](https://vcsxa.egamings.com/wlc/wlc-engine/-/tree/master/src/modules/core/components/feedback-form)

## Параметры компонента:

* `config` - Конфиг, описывающий конкретную форму.
```typescript
export const profileForm: IFormWrapperCParams = {
    class: 'wlc-form-wrapper',
    components: [
        {
            name: 'core.wlc-input',
            params: {
                common: {
                    placeholder: 'Email',
                },
                locked: true,
                name: 'email',
                validators: ['required', 'email'],
                exampleValue: 'dasha.kot@egamings.com',
            },
        },
        {
            name: 'core.wlc-button',
            params: {
                name: 'submit',
                common: {
                    text: 'Save changes',
                },
            },
        },
    ],
};
```

* `ngSubmit` - Функция, которая будет вызвана при отправке формы. 
```typescript
public async ngSubmit(form: FormGroup): Promise<boolean> {
    alert('cool');
}
```

* `formData` - Получаемые данные, если нужно заполнить форму уже сформированными данными.
```typescript
[formData]="userProfile"
```

### В качестве примера предоставлена форма профиля:
```typescript
<div wlc-form-wrapper [config]="config" [formData]="userProfile" [ngSubmit]="ngSubmit.bind(this)"></div>
```

## Компонент принимает три параметра:

1. `Config `- набор компонентов и валидаторов. Валидаторы могут быть как глобальными, так и для конкретного поля. Например:
```typescript
{
    class: 'wlc-form-wrapper',
    components: [
            {
                 name: 'core.wlc-input',
                 params: {
                     theme: 'placeholder-shown',
                     common: {
                         placeholder: 'Your name *',
                     },
                     name: 'senderName',
                     validators: ['required'],
                     exampleValue: 'Example: Ivan',
                 },
            },
            {
                 name: 'core.wlc-input',
                 params: {
                     theme: 'placeholder-shown',
                     common: {
                         placeholder: 'Password',
                         type: 'password',
                     },
                     name: 'currentPassword',
                     validators: ['required'],
                 },
            },
            {
                 name: 'core.wlc-input',
                 params: {
                     theme: 'placeholder-shown',
                     common: {
                         placeholder: 'Confirm password',
                         type: 'password',
                     },
                     name: 'confirmPassword',
                     validators: ['required'],
                 },
            },
    ],
    validators: [
        {
               name: 'matchingFields',
               options: ['currentPassword', 'confirmPassword'],
        }
    ]
};
```

2. `ngSubmit` - метод, который будет вызываться по нажатию на кнопку подтверждения. В качестве входящих параметров принимает экземпляр формы. 

3. `formData` - набор полей для авто заполнения формы. 


## Разберем конкретнее конфиг формы:

  В поле `components` перечислены элементы формы, они полностью повторяет **layout - config.**

### **Элемент input:**
```typescript
{
    name: 'core.wlc-input',
    params: {
        common: {
            placeholder: 'Last name',
        },
        name: 'lastName',
        validators: ['required'],
        exampleValue: 'Ivanov',
    },
},
```
`name` - Название компонента;  
`common: { placeholder: 'Last name', },` - Плейсхолдер элемента;  
`params.name` - Имя поля формы, обязательный параметр;  
`validators` - Массив валидаторов, применимых к данному полю. Можно указать просто перечисление валидаторов, либо передать объект с именем и параметром валидатора. Например:
```typescript
{
    name: 'core.wlc-input',
    params: {
        common: {
            placeholder: 'First name',
        },
        name: 'firstName',
        validators: ['required',
            {
                name: 'minLength',
                options: 2,
            },
            {
                name: 'regExp',
                options: new RegExp('^[A-Za-z]$'),
            },
        ],
        exampleValue: 'Ivan',
    },
},
```
`exampleValue` - Значение, которое будет выводится перед тем, как юзер ввел данные, так называемый пример;

### **Элемент select:**
```typescript
{
    name: 'core.wlc-select',
    params: {
        common: {
            placeholder: 'Gender',
        },
        locked: true,
        name: 'gender',
        validators: ['required'],
        options: 'genders',
    },
},
```

Поле                | Описание
--------------------|---------------------
`name`	            | Название компонента;
`common.placeholder`| Плейсхолдер селекта;
`locked`	        | Запрещает при имеющемся значении изменить данные;
`params.name`       |	Название поля в форме, поле является обязательным;
`validators`        |	Смотри элемент input;
`options`           | Заранее заготовленные константы с значениями, объявленные в select - component, на данный момент доступны - ( `currencies`, `countries`, `genders` ) 
  
### **Глобальные валидаторы:**
  Можно установить валидацию на все поля формы, которые будут применяться ко всем. Для этого нужно на уровне конфига формы задать ключ **validators** и указать нужные валидаторы. Пример сравнения двух полей через глобальные валидаторы:

```typescript
validators: [
        {
               name: 'matchingFields',
               options: ['currentPassword', 'confirmPassword'],
        }
    ]
```

В поле **options** нужно вставить имена элементов, которые нужно сравнить.

### Валидаторы:
  Список доступных валидаторов на данных момент. Актуальный список в **validation.service.ts** (core module):

Название            | Описание
--------------------|------------------------------
`emailUnique`       | Проверка уникальности email;
`loginUnique`       | Проверка уникальности login;
`password `         | Соответствие пароля сложности;
`required`          | Обязательность поля;
`minLength `        | Минимальная длина поля;
`maxLength`         | Максимальная длина поля;
`regExp`            | Соответствие значения выбранному паттерну;
`matchingFields`    | Совпадение значения двух полей;
`email`             | Проверка на корректность написания email;
