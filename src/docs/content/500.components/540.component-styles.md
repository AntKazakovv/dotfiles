# 5.4. Стили компонента

## Структура директории styles

* **![](../../assets/folder.svg) themes** - _директория с файлами тем компонента. Для каждой темы - минимум отдельный
  SCSS файл. Для сложных компонентов можно использовать любое необходимое кол-во вложенных директорий. Главное правило:
  название директории должно семантически соответствовать её содержимому._
    * ![](../../assets/folder.svg) \_defaut.scss - _тема по умолчанию. Должна быть для каждого компонента, так как если
      не передать никаких параметров в компонент то ему автоматически применяется тема default._
    * ![](../../assets/folder.svg) \_rounded.scss - _(пример: тема rounded)._
    * ![](../../assets/folder.svg) \_others.scss - _(пример: тема others)._
* **![](../../assets/folder.svg) types** - _Стили для типов компонента, если требуется. В большинстве случаев типы
  описываются в файлах темы._
    * ![](../../assets/folder.svg) \_defaut.scss
    * ![](../../assets/folder.svg) \_rounded.scss
    * ![](../../assets/folder.svg) \_others.scss
* **![](../../assets/folder.svg) utils** - _обязательная директория с дополнительными файлами стилей компонента._
    * ![](../../assets/folder.svg) \_mixins.scss - _(не обязательно) локальные миксины компонента._
    * ![](../../assets/folder.svg) \_functions.scss - _(не обязательно) локальные функции компонента._
    * ![](../../assets/folder.svg) \_vars.scss - _локальные переменные компонента._
* scratch.component.scss - _точка входа. Именно этот файл указывается в TS файле компонента как файл стилей._

## scratch.component.scss

```scss
@import './themes/default';
@import './themes/rounding';
```

Представляет собой индексный файл, импортирующие необходимые темы и типы. Так же может включать в себя глобальные стили
компонента, не привязанные к теме, например сброс стилей.

## utils/_vars.scss

```scss
@use 'engine-styles/functions' as ef;
 
// Size
$minWidth: 100px !default;
$heightSmall: 35px !default;
$heightDefault: 40px !default;
$heightLarge: 50px !default;
 
// Font
$fontSize: ef.get('font', 'main') !default;
$fontColor: ef.get('color', 'text') !default;
$fontColorActive: ef.get('color', 'text') !default;
$fontColorSecondary: ef.get('color', 'main') !default;
$fontColorActiveSecondary: ef.get('color', 'text') !default;
 
// Transform
$textTransform: uppercase !default;
```

Локальные переменные компонента. Важно: в других файлах стилей не должны использоваться глобальные переменные, только
локальные.

Если возникает необходимость в разных переменных для разных типов/тем, необходимо использовать соответствующие префиксы
и выделять переменные в отдельные блоки.

Блоки переменных необходимо называть осмысленно.

## themes/_default.scss

```scss
@import '../utils/mixins';
@import '../utils/vars';
 
.wlc-btn--theme-default {
 
    &:host {
        @include buttonBase;
 
        border-radius: 2px;
        border: unset;
 
        &:hover {
            background: $backgroundHover;
        }
 
        &.wlc-btn--theme-mod-secondary {
            @include buttonTransparency;
        }
    }
}
```

Стили для темы обязательно должны инкапсулироваться внутри соответствующего модификатора. Необходимо для исключения
влияния стилей одной темы на другую.

Все пересекающиеся между темами блоки SCSS кода необходимо выносить в миксины. Это даст гибкость в работе и исключит
повторяемость кода.

Так как родительский элемент находится по умолчанию вне скоупа стилей используется :host.
