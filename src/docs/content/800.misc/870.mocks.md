# 8.7. Работа с mock-данными

## Подключение mock-данных на проект
Mock-данные подключаются в файле [01.base.config.ts]().
Сборка проекта с mock-данными осуществляется стандартным способом.
По умолчанию mock-данные для запросов отключены.

### Интерфейс подключения
```typescript
interface IMocksConfig {
    base?: {
        jackpots?: boolean;
        wins?: boolean;
        topWins?: boolean;
        bonuses?: boolean;
        tournaments?: boolean;
    },
    custom?: [ICustomMock, ...ICustomMock[]],
}

interface ICustomMock {
    route: string;
    source: string;
}

```

## Подключение дефолтных mock-данных для запросов

Для интересующих запросов в файле **01.base.config.ts** в `$base.mocks.base` надо поставить значение ключа: `true`.

Таким образом для указанных запросов будут подключены дефолтные mock данные.

Дефлотные mock-данные представлены в виде **.json** файлов и находятся в [src/system/mocks/](https://wlcgitlab.egamings.com/wlc/wlc-engine/-/tree/master/src/system/mocks)

### Пример подключения:
```typescript
export const $base: IBaseConfig = {
    mocks: {
        base: {
            jackpots: true,
            wins: true,
            topWins: true,
            bonuses: true,
            tournaments: true,
        },
    },
};
```
## Подключение своих собственных mock-данных для конкретных запросов

1. В папке [/static/](), создаем новый **.json** файл, который будет выступать в качестве ответа на запрос.
2. На проекте в `$base.mocks.custom` в массиве из объектов в `route` прописываем api запросы, а в `source` пути до файлов с mock-данными.

### Пример:

```typescript
export const $base: IBaseConfig = {
    mocks: {
        custom: [
            {
                route: '/api/v1/profiles',
                source: '/static/mock-profile.json',
            },
        ],
    },
};
```
