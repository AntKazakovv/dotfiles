# 8.4. Favicon
## How to include
Favicons can be included in the `<header>` section of *index.html* in the following way:  
```html
{% if 'template/additional-header.html' is ondisk %}
    {% include 'additional-header.html' %}
{% else %}
    {% if 'favicon/' is ondisk %}
        <link rel="apple-touch-icon" sizes="180x180" href="/favicon/apple-touch-icon.png">
        <link rel="icon" type="image/png" sizes="32x32" href="/favicon/favicon-32x32.png">
        <link rel="icon" type="image/png" sizes="192x192" href="/favicon/android-chrome-192x192.png">
        <link rel="icon" type="image/png" sizes="16x16" href="/favicon/favicon-16x16.png">
        <link rel="manifest" href="/favicon/site.webmanifest">
        <link rel="shortcut icon" href="/favicon/favicon.ico" type="image/x-icon"/>
        <meta name="msapplication-TileColor" content="#f17105">
        <meta name="msapplication-TileImage" content="mstile-144x144.png">
        <meta name="theme-color" content="#000000">
        <meta name="apple-mobile-web-app-capable" content="yes">
        <meta name="apple-mobile-web-app-status-bar-style" content="#000000">
        <meta name="msapplication-TileColor" content="#000000">
        <meta name="msapplication-config" content="/favicon/browserconfig.xml" />
    {% else %}
        <link rel="shortcut icon" href="/favicon.ico" type="image/x-icon"/>
    {% endif %}
{% endif %}
```
To include a custom favicon, create the *additional-header.html* file in the [/roots/template/]() folder.
## Structure
The project's favicon is stored in [/roots/favicon/](). 
The recommended structure for the proper display on most browsers and platforms is as follows:

Файл                        | Назначение         | Размеры
----------------------------|--------------------|------------------
favicon.ico                 | Legacy IE10 and below | 16x16 32x32 48х48    
favicon-16x16.png           | Classic desktop    | 16x16   
favicon-32x32.png           | Standard desktop   | 32x32 
android-chrome-192x192.png  | Android / Chrome   | 192x192
android-chrome-512x512.png  | Android / Chrome   | 512x512
mstile-150x150.png          | Win8+ metro        | 270х270px (!)   
apple-touch-icon.png        | Apple devices      | 180x180px  
safari-pinned-tab.svg       | Apple safari       | Vector 
browserconfig.xml           | Win config         |
site.webmanifest            | Android config     |

In addition, make sure to copy *favicon.ico* to [/roots/](), since some services and legacy web browsers may search for the favicon in the root directory.
## Images
Transparent backgrounds are highly recommended. You can set up the background color in the configuration.  

If the icon is distorted or blurred when scaling, consider asking your web designer to create a separate version with the required resolution.
## Конфигурация
**[browserconfig.xml]()** contains instructions for the proper display of the website tile when the "Pin website to start menu" feature is used in Windows 8 and above with IE11/Edge browsers:
```xml
<browserconfig>
    <msapplication>
        <tile>
            <square150x150logo src="/mstile-150x150.png"/>
            <TileColor>#da532c</TileColor>
        </tile>
    </msapplication>
</browserconfig>
```
where `TileColor` is the tile's background color.  

**[site.webmanifest]()** is a manifest JSON file for the icons to be used on the website for Android/Chrome, which is defined by the W3C and maintained by Google. It has the following format:
```json
{
    "name": "DevCasino",
    "short_name": "DevCasino",
    "start_url": "/",
    "icons": [
        {
            "src": "/favicon/android-chrome-192x192.png",
            "sizes": "192x192",
            "type": "image/png"
        },
        {
            "src": "/favicon/android-chrome-512x512.png",
            "sizes": "512x512",
            "type": "image/png"
        }
    ],
    "theme_color": "#ffffff",
    "background_color": "#ffffff",
    "display": "standalone"
}

```
`"display": "standalone"` - the website is displayed in a separate window in the browser without the standard UI elements such as the URL bar.     
`theme_color` - sets the toolbar's color.  
`background_color` - is used on the splash screen when the website is displayed on a mobile device.

## Картинки
Прозрачность фона приветствуется. Цвет фона может задаваться через конфиг.  

Если при масштабировании иконка теряет вид или замыливается - стоит попросить дизайнера подготовить вам отдельную версию для нужного разрешения.
## Конфигурация
**[browserconfig.xml]()** - инструкции для правильного отображения плитки сайта при использовании функции «Закрепить сайт на начальном экране» в Windows 8 и выше с браузерами IE11/Edge:
```xml
<browserconfig>
    <msapplication>
        <tile>
            <square150x150logo src="/mstile-150x150.png"/>
            <TileColor>#da532c</TileColor>
        </tile>
    </msapplication>
</browserconfig>
```
где `TileColor` - цвет фона плитки.  

**[site.webmanifest]()** - манифест JSON-файл, определенный W3C и поддерживаемый Google, в котором указывают иконки для сайта для Android/Chrome. Имеет следующий формат:
```json
{
    "name": "DevCasino",
    "short_name": "DevCasino",
    "start_url": "/",
    "icons": [
        {
            "src": "/favicon/android-chrome-192x192.png",
            "sizes": "192x192",
            "type": "image/png"
        },
        {
            "src": "/favicon/android-chrome-512x512.png",
            "sizes": "512x512",
            "type": "image/png"
        }
    ],
    "theme_color": "#ffffff",
    "background_color": "#ffffff",
    "display": "standalone"
}

```
`"display": "standalone"` - сайт запускается в собственном окне, отдельном от браузера, и скрывает стандартные элементы пользовательского интерфейса браузера, такие как строка URL.     
`theme_color` - устанавливает цвет панели инструментов.  
`background_color` - используется на экране-заставке при запуске сайта на мобильном устройстве.
