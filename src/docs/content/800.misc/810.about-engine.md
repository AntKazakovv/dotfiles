# 8.1.0. Знакомство с проектом на основе WLC Engine

Итак, перед Вами стандартная структура проекта на WLC Engine:

![Drag Racing](../../../assets/about-engine/about-engine1.png)  

Глобальные настройки проекта расположены  в папке **config**, где размещены настройки для бекенда и фронтенда проекта в соответствующих папках. Нас пока интересуют папка **frontend**, которая представлена данными файлами:

![Drag Racing](../../../assets/about-engine/about-engine2.png)  

## **01.base.config:**  
Содержит общие настройки проекта, такие как название проекта, контакты, используемые разделы сайта. Все дефолтные настройки engine находятся по адресу - [wlc-engine/modules/core/system/config/base/index.ts]()

```typescript
import {IBaseConfig} from 'wlc-engine/modules/core';
 
export const $base: IBaseConfig = {
    app: {
        type: 'wlc',
    },
    site: {
        name: 'DevCasino',
        url: '',
    },
    contacts: {
        phone: '+78001234567',
        email: 'test@test.com',
    },
    profile: {
        store: {
            use: true,
        },
    },
};
```

## **02.layout.config:**
Содержит настройки проекта по стейтам, если конфиг пустой, то настройки полностью наследуются из **engine**. Все дефолтные настройки **layout** находятся по адресу - [wlc-engine/modules/core/system/config/layouts/index.ts]()

Пример кастомного конфига из проекта:
```typescript
import { ILayoutsConfig } from 'wlc-engine/modules/core/system/interfaces';
import * as componentLib from 'wlc-engine/modules/core/system/config/layouts/components';
import * as sectionsLib from 'wlc-engine/modules/core/system/config/layouts/sections';
 
 
export const $layouts: ILayoutsConfig = {
    'app.home': {
        replaceConfig: false,
        sections: {
            'content-games': {
                modify: [
                    {
                        type: 'delete',
                        position: 2,
                    },
                ],
            },
            'promo': {
                container: true,
                components: [
                    {
                        name: 'core.wlc-wrapper',
                        params: {
                            class: 'wlc-home-winners',
              componentName                      params: {
                                        type: 'latest',
                                        title: 'Last winners',
                                        theme: 'vertical',
                                        themeMod: 'vertical',
                                    },
                                },
                            ],
                        },
                    },
                ],
            },
            'content-games-2': {
                container: true,
                components: [
                    componentLib.wlcGamesGrid.roullete1row,
                ],
            },
        },
    },
}; 
```
## **03.files.config:**
Локальные файлы, которые попадут в сборку проекта. Все дефолтные файлы из engine находятся по адресу - [wlc-engine/modules/core/system/config/files.config.ts]()

## **04.modules.config:**
Глобальные настройки модулей и компонентов в проекте. Дефолтные настройки заданы на уровне компонентов.

Ключ - название модуля из компонента (указывается в `moduleName` поле в `defaultParams` компонента). Нужно также не забыть передать в `super configService`. 

Название компонента указывается там же в `componentName`. Настройки из данного конфига мержатся с дефолтными настройками компонента.  

Пример кастомного конфига из проекта:
```typescript
export const $modules = {
    core: {
        config: {},
        components: {
            'wlc-button': {
                theme: 'rounding',
            },
        },
    },
    games: {
        components: {
        },
    },
    static: {
        config: {},
    },
};
```
Вся основная работа над проектом ведется в папке src, которая имеет структуру:  
![Drag Racing](../../../assets/about-engine/about-engine3.png)   

Подробнее разберем по папкам:  
![Drag Racing](../../../assets/about-engine/about-engine4.png)  

**_app.variables** - файл с кастомными глобальными переменными, которые мержатся с переменными из  **engine**. Глобальные переменные можно посмотреть по адресу - [wlc-engine/src/engine-scss/_engine.variables.scss]()

Пример использования кастомных переменных:
```typescript
$font-family-sans-serif: 'Noto Sans', Arial, sans-serif;
 
$customBaseVariables: (
    'imageBackground': #101010 url('/static/images/site-bg.jpg') no-repeat center bottom,
);
 
$customMainColors: (
    'main': #ff3d00,
    'secondary': #c5b358,
    'buttonColor': #f94b00,
    'baseOpacity': #fff,
    'bg100': #bebebf,
    'bg300': #603a02,
    'bg500': #6a3208,
    'bg700': #35251e,
    'bg': #101010,
);
```
**_app.styles** - по умолчанию пустой файл стилей для проекта, куда можно подключать кастомные стили, если они понадобятся. Пример использования - создание папки **project-styles**, куда по разделам сайта складывать стили, импортируя их в **app.styles**. Чем продуманнее структура - тем лучше. **НЕ ЗАБЫВАЕМ, ЧТО ДЛЯ ЦВЕТОВ САЙТА ПО ФАЙЛАМ И КОМПОНЕНТАМ ИСПОЛЬЗУЕМ SCSS ПЕРЕМЕННЫЕ.** В будущем это поможет без проблем поддерживать проект и менять его, не затрачивая много времени. 

## **Custom:**
![Drag Racing](../../../assets/about-engine/about-engine5.png)

Данная папка служит  для настройки компонентов проекта, где по модулям распределены компоненты из движка. Для работы с ними нужно в названии файлов убрать тильду ~ и перезапустить сборку, только после этого файлы попадут в сборку. Для кастомизации представлены только **scss** и **html** файлы, **ts** нельзя изменить в проекте.  Сама папка генерится автоматически при сборке проекта.

Для работы со стилями, нужно зайти в файл стилей, ***перед*** импортом стилей объявить мап переменных, которые нужно переопределить, после этого данные переменные будут  применены к компоненту в движке.

![Drag Racing](../../../assets/about-engine/about-engine6.png)

Сами кастомные стили пишем **ПОСЛЕ** импорта.
