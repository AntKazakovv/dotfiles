# 6.1. Config service (WIP)

The configuration service is the core configuration service of the application: it starts up **bootstrap** and generates the global configuration of the application. In addition, it provides the **get** and **set** methods for obtaining and specifying its values, respectively.

## How to use the "get" method
This method accepts either a `string` containing the path to the target value (`'appConfig.mobile'`), or an object containing extended parameters (`getParams`).

### getParams
Field            | Interface       | Value
----------------| ----------------| ---------------------------
`name`            | string          | The path to the target value;
`storageType `   | IStorageType     | The type of the storage from which to retrieve the information: `type IStorageType = 'localStorage' \| 'sessionStorage' \| 'cookiesStorage';`

**`get\<T\>(value)`** - when retrieving data through a generic, make sure to specify the expected type; otherwise, the method will return `unknown`. Example:

```typescript
this.ConfigService.get<boolean>('appConfig.mobile');
```

# 6.1. Config service (WIP)

  Конфиг сервис представляет собой главный конфигурационный сервис приложения, он загружает **bootstrap** и формирует глобальный конфиг приложения.
  Имеет методы **get** и **set** для получения и установки значений.

## Работа с методом get:
  Метод принимает на вход либо **string** с путем до искомого значения **('appConfig.mobile)**, либо объект с тонкой настройкой **getParams.**

### getParams
Поле            | Интерфейс       | Значение
----------------| ----------------| ---------------------------
`name`          | string          | Путь до искомого значения;
`storageType `  | IStorageType     | Тип хранилища, откуда нужно получить информацию: `type IStorageType = 'localStorage' \| 'sessionStorage' \| 'cookiesStorage';`

**`get\<T\>(value)`** - при получении данных через дженерик мы должны указать тип, который ожидаем, иначе вернется **unknown**. Пример:

```typescript
this.ConfigService.get<boolean>('appConfig.mobile');
```
