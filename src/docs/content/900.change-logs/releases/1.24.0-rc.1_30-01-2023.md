### SCR [#391785](https://tracker.egamings.com/issues/391785) - chore: Added domain balancer

This functionality completely repeats the functionality described in the [theme](https://wlcgitlab.egamings.com/wlc/angular-wlc-theme-bootstrap4/-/merge_requests/3005) and the [gulp-casino](https://wlcgitlab.egamings.com/wlc/gulp-casino/-/merge_requests/78). When building the project, the `~domainBalancer.ts` file is copied to the src/system/inline directory. To make the script work, you need to remove the `"~"` prefix and enter the required `mainDomains` and `mirrors`. After saving the changes, a compiled js file containing the minified script from the `domainBalancer.ts` file will be created in the roots/template directory. JS file is already registered in the project template (`head.tpl`). Gulp watches for changes in the `domainBalancer.ts` file. and automatically rebuilds the js file.

**UPD**: Before compiling the ts file to js, ​​you must manually update the configs on the project through the command  `gulp update:configs`.

### SCR [#419224](https://tracker.egamings.com/issues/419224) - chore: Added gulp task for check captcha keys

This gulp-task, during the build of the project, checks for the presence of the keys necessary for the captcha to work. How it works: if the project config contains the `recaptcha = true`, the task will check for the presence of the `recaptchaSiteKey` and `recaptchaSecretKey` parameters. If at least one of these two parameters is missing, or if the keys are empty, the project build process stops and an error is thrown into the console. `ERROR: No value was set for recaptchaSiteKey or recaptchaSecretKey.`
