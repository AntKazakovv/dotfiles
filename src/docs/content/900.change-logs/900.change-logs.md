Определение правил происходит по ключу поля.

Если задан дефолтный конфиг **default** то данное правило будет применяться к любому полю, чей ключ соответствует конфигу.

Если необходимо задать дополнительное правило, которое должно применяться только для конкретной платежки/платежек и/или для пользователей с конкретной страной/странами, то необходимо использовать массив **settings**. Платежки указываются массивом алиасов в поле systems, а страны массивом в поле countries (iso alpha3).

**Внимательно** проверяйте ключи полей т.к. лейблы могут одинаковые, а ключи разные, например
- accountNumber: {label: "Account number", optional: "1", showfor: "withdraw"}
- account_number: {label: "Account number", optional: "1", showfor: "withdraw"}
- acct: {label: "Account number", showfor: "withdraw", skipsaving: "1"}

Также **внимательно** проверяйте алиасы платежных систем т.к. pagsmile и pagsmile_brazil для ПМов, тестировщиков и саппорта кажутся одной и тойже системой, а для нас это принципиально разные.
