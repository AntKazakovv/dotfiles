# 3.2.2. Конфигурация слоёв

**Layout config** представляет собой объект с конфигурацией сайта по стейтам. Полями объекта являются стейты приложения.
На старте загрузки страницы проверяется нынешний стейт и применяются конкретные настройки. Первое поле **‘app’** –
является структурным, конкретной реализации этого стейта нет, он нужен для указания типового стейта приложения.

## Настройки стейта

Поле            | Интерфейс | Описание
--------------- | --------- | --------
replaceConfig?  | boolean | Заменить ли дефолтный конфиг из движка, по умолчанию false, происходит мерж с дефолтным конфигом;
subcategories?  | [ILayoutsConfig](../../../interfaces/ILayoutsConfig.html) | представляет собой тонкую настройку стейта с параметрами, по примеру catalog/:category, где category – представляет собой параметр, по дефолту отрабатывает с двумя параметрами, slug или category. Содержимое поля subcategories полностью повторяет IlayoutsConfig;
sections?       | [ILayoutSectionsConfig](../../../interfaces/ILayoutSectionsConfig.html) | перечислений секций, которые будут представлены на странице, название секции будет фигурировать в классе секции, остальное название произвольно;
extends?        | string | Наследовать ли другой стейт, подтянет секции из другого стейта;

Настройки самих секции вынесены в отдельные файлы. Расположены в src/modules/core/system/config/layouts/sections.

Файл с настройками секции может включать несколько версий одной секции. Версии секции объединяются через неймспейс в
формате: namespace <ИМЯ СЕКЦИИ>

**Пример**

```typescript
export namespace profileContent {
    export const profileDeposit: ILayoutSectionConfig = {
        container: true,
        components: [
            componentLib.wlcDepositWithdraw.deposit,
        ],
    };

    export const profileWithdraw: ILayoutSectionConfig = {
        container: true,
        components: [
            componentLib.wlcDepositWithdraw.withdraw,
        ],
    };
}
```

В основном конфиге, секцию из примера можно подключить так: **sectionsLib.profileContent.profileDeposit** или
**sectionsLib.profileContent.profileWithdraw**.

## Настройка конкретной секции:

Поле            | Интерфейс | Описание
--------------- | ----------| --------
container?      | string or boolean | Применить ли класс container для секции, либо конкретный класс;
components?     | \([ILayoutComponent](../../../interfaces/ILayoutComponent.html) or string\)\[\] | Массив из объектов компонентов или строк названий компонентов;
modify?         | [ILayoutModifyItem](../../../interfaces/ILayoutModifyItem.html)\[\] | Модификация компонента – способ модификации конфига точечно, имеет данные поля, можно добавить, заменить или удалить конкретный компонент или изменить его конфигурацию из секции.
modifiers?      | string\[\] | Список модификаторов для секции;
theme?          | string | Используемая тема для компонента;
order?          | number | Сортировка секции в DOM, по умолчанию все секции имеют вес 1;

### Пример modify

```typescript
modify: [
    {
        type: 'replace', // замена компонента
        position: 'base.wlc-logo#2',
        component: {
            name: 'base.test-replace'
        },
    },
    {
        type: 'insert', // вставка компонента
        position: 'before base.wlc-logo#2',
        component: {
            name: 'base.test-insert'
        }
    },
    {
        type: 'delete',  // удаление компонента
        position: 'base.wlc-logo#2',
    },
    {
        type: 'merge', // изменение параметров существующего компонента
        position: 0,
        component: {
            params: {
                wlcElement: 'new-wlc-element',
            },
        },
    },
]
```

Для удаления секции достаточно указать ее в конфиге как null
**Пример**

```typescript
'app'
:
{
    sections: {
        footer: null,
    }
,
}
,
```

Конфиги компонентов вынесены в отдельные файлы по аналогии с секциями. Расположены в папке
src/modules/core/system/config/layouts/components и сгруппированы по модулям.
**Пример**
```typescript
export namespace wlcBonusesList {
    export const main: ILayoutComponent = {
        name: 'bonuses.wlc-bonuses-list',
        params: {
            common: {
                title: '',
                filter: 'main',
            },
        },
    };
    export const active: ILayoutComponent = {
        name: 'bonuses.wlc-bonuses-list',
        params: {
            common: {
                title: '',
                restType: 'active',
                filter: 'active',
            },
        },
    };
    export const inventory: ILayoutComponent = {
        name: 'bonuses.wlc-bonuses-list',
        params: {
            common: {
                title: '',
                filter: 'inventory',
            },
        },
    };
}
```

## Настройка компонента:

Поле            | Интерфейс | Описание
--------------- | ----------| --------
name            | string    | Название компонента, содержит в себе два поля, `${moduleName}.${nameComponent}`. Первым в имени компонента идет название модуля, вторым в названии компонента идет имя, экспортируемое из модуля. Пример: 'menu.wlc-menu';
componentClass? | string    | Класс компонента (сам инстанс компонента);
injector?       | Injector  | Передача инжектора в компонент;
params?         | unknown   | Конкретные параметры компонента, в конфиге нужно указывать, чтобы можно было получить все доступные поля. Параметры компонента лучше всего писать подробнее, без сокращений.
display?        | IComponentDisplayConfig | Настройка отображение компонента, имеет поля: <br> **after** {_number_} -  display component after this window width; <br> **before** {_number_} -  display component before this window width; <br> **mobile** {_boolean_} -  display component only on mobile when true, on desktop when false; <br> **auth** {_boolean_} -  display component only auth user when true, anon user when false; 
exclude?        | string\[\] | Исключить компонент из отображения на указанных стейтов;
include?        | string\[\] | Отображать компонент только на указанных стейтах;

**Пример конфига из движка:**
```typescript
export const $layouts: ILayoutsConfig = {
    'app': {
        replaceConfig: true,
        sections: {
            header: sectionsLib.header.def,
            footer: sectionsLib.footer.def,
        },
    },
    'app.home': {
        replaceConfig: true,
        extends: 'app',
        sections: {
            'banner-section': sectionsLib.bannerSection.home,
            'categories': sectionsLib.categories.catalog,
            'content-games': sectionsLib.contentGames.home,
        },
    },
    'app.gameplay': {
        extends: 'app',
        sections: {
            content: sectionsLib.content.gamePlay,
        },
    },
    'app.profile': {
        extends: 'app',
        sections: {
            'profile-menu': sectionsLib.profileMenu.def,
        },
    },
    'app.profile.main.info': {
        extends: 'app.profile',
        sections: {
            'profile-content': sectionsLib.profileContent.profileMain,
        },
    },
}
```

## Кастомизация PanelLayout

PanelLayout - отдельный конфиг для боковых меню. Кастомизируется аналогичным способом. В проекте в папке config/frontend необходимо создать файл, который экспортирует переменную $panelsLayouts с конфигом (например 05.panels-layouts.config.ts). По умолчанию файла в проекте нет.

Необходимо убедиться, что конфиг экспортируется в index.ts.

```typescript
export {$panelsLayouts} from './05.panels-layouts.config';
```

## Переключение типа проекта

Предполагается несколько вариантов лайаутов для разных типов проектов. Например: wlc и affiliate.

Для быстрого переключения типа проекта нужно в конфиге $base проекта в секции app указать type. В данный момент доступны типы wlc и aff. По умолчанию устанавливается wlc

**Пример**

```typescript
export const $base: IBaseConfig = {
    app: {
        type: 'wlc',
    },
    site: {
        name: 'BestChance',
        url: '',
    },
};
```
