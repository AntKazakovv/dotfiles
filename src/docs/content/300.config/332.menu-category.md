#  3.3.2. Меню категорий

## Добавление разделов в основное меню
  По умолчанию в проекте создается два раздела: **Casino** и **LiveCasino**.
  Это прописано дефолтными настройками в движке (на стороне **fundist** ничего настраивать не нужно).

## Создание дочерних категорий для каждого раздела
  Формируется автоматически. По умолчанию дробится только раздел **Casino** (на стороне **fundist** ничего настраивать не нужно)

## Создание родительских категорий
  Указываем по ключу `parents` слаги категорий, которые должны быть разделами (по умолчанию это разделы **casino** и **livecasino**).  
  Данные категории будут опубликованы в основном меню. По умолчанию раздел casino раздробится на дочерние категории, которые будут опубликованы в меню категорий.  
  Если из списка убрать **casino**, то из основного меню он будет убран, но продолжит существовать как раздел (так как он по умолчанию категория всегда родительская, это специфическая особенность для **casino**) и его дочерние категории будут в меню категорий. Для того чтобы избавиться от **casino**, см. ниже исключение категорий.
```typescript
import {IGamesConfig} from '../interfaces/games.interfaces';
 
export const $games: IGamesConfig = {
    categories: {
        parents: [
            'casino',
            'livecasino',
        ],
    },
};
```

### Исключение категорий
  Если заранее известно, что некоторые категории вовсе не нужны, их по умолчанию можно убрать используя `exclude.buSlug`.  
  Например, избавимся от категории **casino**:
```typescript
import {IGamesConfig} from '../interfaces/games.interfaces';
 
export const $games: IGamesConfig = {
    categories: {
        exclude: {
            bySlug: [
                'casino',
            ],
        },
    },
};
```
  При этом, так как по умолчанию существовало две родительские категории **casino** и **livecasino**, в основном меню останется только **livecasino**.  
  И **livecasino**, по аналогии с разделом **casino**, будет по умолчанию раздроблен на дочерние категории, которые будут опубликованы в меню категорий.  
  Если у категории отсутствует `slug`, то в качестве `slug` используется `menuid`.

## Практические примеры
**На сайте нужен только раздел Livecasino**
  Для этого нужно исключить категорию **casino** через `exclude.bySlug`. Тогда она перестанет сущестовать как категория и будет убрана из основного меню.   
  При использовании **ключа `parents`** раздел казино был бы убран из основного меню, но при этом бы продолжил существовать как родительская категория и 
  его дочерние категории были бы опубликованы в меню категорий.
```typescript
import {IGamesConfig} from '../interfaces/games.interfaces';
 
export const $games: IGamesConfig = {
    categories: {
        exclude: {
            bySlug: [
                'casino',
            ],
        },
    },
};
```
