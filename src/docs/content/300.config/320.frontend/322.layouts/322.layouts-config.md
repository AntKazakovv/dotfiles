# 3.2.2. Layout configuration

**Layout config** s the object with the website configuration by state. The object's field contains application states. When a webpage is loading, the system checks the current state and applies specific settings. The first field, `‘app’`, is the structural field, which does not have any specific implementation of this state and is used to indicate a typical application state.

## State settings

Field            | Interface | Description
--------------- | --------- | --------
`replaceConfig?`  | boolean | Whether to replace the default configuration of the engine (false by default); it is merged with the default configuration;
`subcategories?`  | [ILayoutsConfig](../../../interfaces/ILayoutsConfig.html) | It is a fine-tuning of a state with parameters, such as catalog/:category, where "category" is one of the two default parameters ("slug" and "category"). The content of the "subcategories" field is exactly the same as of `IlayoutsConfig`;
`sections?`       | [ILayoutSectionsConfig](../../../interfaces/ILayoutSectionsConfig.html) | It is an enumeration of sections that will be present on the webpage; the section's name will appear in the section's class, while all other names are arbitrary;
`extends?`        | string | Whether to inherit another state; the sections are retrieved from another state;

The settings of the sections themselves are stored in separate files. They are available in **src/modules/core/system/config/layouts/sections**.

A file containing the section settings may include several versions of the same section. Section versions are combined through a namespace in the following format: namespace `<_SECTION NAME_>`

**Example**

```typescript
export namespace profileContent {
    export const profileDeposit: ILayoutSectionConfig = {
        container: true,
        components: [
            componentLib.wlcDepositWithdraw.deposit,
        ],
    };

    export const profileWithdraw: ILayoutSectionConfig = {
        container: true,
        components: [
            componentLib.wlcDepositWithdraw.withdraw,
        ],
    };
}
```

You can add a section from the example to the basic configuration in the following way:
`sectionsLib.profileContent.profileDeposit` or
`sectionsLib.profileContent.profileWithdraw`.

## The settings of a given section are as follows:

Field            | Interface | Description
--------------- | ----------| --------
`container?`      | string or boolean | Whether to apply the "container" class to the section, or a specific class;
`components?`     | \([ILayoutComponent](../../../interfaces/ILayoutComponent.html) or string\)\[\] | An array of component objects or an array of strings containing component names;
`modify?`         | [ILayoutModifyItem](../../../interfaces/ILayoutModifyItem.html)\[\] | The component's modification is a method for modifying specific elements of the configuration: you can add, replace or delete a specific component or modify its configuration from the section.
`modifiers?`      | string\[\] | List of modifiers for the section;
`theme?`          | string | The theme used for the component;
`order? `         | number | Sorting the section in the DOM; all sections have a weight of 1 by default;

### Example of modify

```typescript
modify: [
    {
        type: 'replace', // replacing the component
        position: 'base.wlc-logo#2',
        component: {
            name: 'base.test-replace'
        },
    },
    {
        type: 'insert', // inserting the component
        position: 'before base.wlc-logo#2',
        component: {
            name: 'base.test-insert'
        }
    },
    {
        type: 'delete',  // deleting the component
        position: 'base.wlc-logo#2',
    },
    {
        type: 'merge', // modifying the parameters of the existing component
        position: 0,
        component: {
            params: {
                wlcElement: 'new-wlc-element',
            },
        },
    },
]
```

To delete a section, simply specify it as `null` in the configuration
**Example:**

```typescript
'app'
:
{
    sections: {
        footer: null,
    }
,
}
,
```

Component configurations are placed in separate files, just like sections. They are stored in **src/modules/core/system/config/layouts/components** and grouped by modules.
**Example:**
```typescript
export namespace wlcBonusesList {
    export const main: ILayoutComponent = {
        name: 'bonuses.wlc-bonuses-list',
        params: {
            common: {
                title: '',
                filter: 'main',
            },
        },
    };
    export const active: ILayoutComponent = {
        name: 'bonuses.wlc-bonuses-list',
        params: {
            common: {
                title: '',
                restType: 'active',
                filter: 'active',
            },
        },
    };
    export const inventory: ILayoutComponent = {
        name: 'bonuses.wlc-bonuses-list',
        params: {
            common: {
                title: '',
                filter: 'inventory',
            },
        },
    };
}
```

## Component's settings:

Field           | Interface | Description
--------------- | ----------| --------
`name`            | string    | The component's name contains two fields: `${moduleName}.${nameComponent}`. The first part of the component's name is the module's name and the second part is the name exported from the module. Example: 'menu.wlc-menu';
`componentClass?` | string    | Class of the component (the component's instance itself);
`injector?`       | Injector  | Provision of the injector to the component;
`params?`         | unknown   | The specific parameters of the component must be specified in the configuration, so that all the available fields could be retrieved. The component's parameters should be named clearly and without abbreviations.
`display?`        | IComponentDisplayConfig | The component's appearance settings include the following fields: <br> **after** {*number*}: display the component to the right of this window; <br> **before** {*number*}: display the component to the left of this window; <br> **mobile** {*boolean*}: display the component only on mobile devices when true or on desktop PCs when false; <br> **auth** {*boolean*}: display the component only for authenticated users when true or for anonymous users when false;
`exclude?`        | string\[\] | Do not display the component for the specified states;
`include?`       | string\[\] | Display the component only for the specified states;

**Example from the engine's configuration:**
```typescript
export const $layouts: ILayoutsConfig = {
    'app': {
        replaceConfig: true,
        sections: {
            header: sectionsLib.header.def,
            footer: sectionsLib.footer.def,
        },
    },
    'app.home': {
        replaceConfig: true,
        extends: 'app',
        sections: {
            'banner-section': sectionsLib.bannerSection.home,
            'categories': sectionsLib.categories.catalog,
            'content-games': sectionsLib.contentGames.home,
        },
    },
    'app.gameplay': {
        extends: 'app',
        sections: {
            content: sectionsLib.content.gamePlay,
        },
    },
    'app.profile': {
        extends: 'app',
        sections: {
            'profile-menu': sectionsLib.profileMenu.def,
        },
    },
    'app.profile.main.info': {
        extends: 'app.profile',
        sections: {
            'profile-content': sectionsLib.profileContent.profileMain,
        },
    },
}
```

## Customization of PanelLayout

PanelLayout - is a separate configuration for sidebars. It can be customized similarly. In the config/frontend folder of your project, create a new file that exports the $panelsLayouts variable with the configuration (e.g. **05.panels-layouts.config.ts**). By default, projects do not contain such file.

Make sure that the configuration is exported to **index.ts**.

```typescript
export {$panelsLayouts} from './05.panels-layouts.config';
```

## Switching among the project types

There are several options for layouts depending on the project type. Example: **wlc** and **affiliate**.

To quickly switch among the project types, you need to specify "type" in the `$base` configuration of the project in the "app" section. Currently, only two projects types are available: `wlc` and `aff`. The default type is `wlc`.

**Пример**

```typescript
export const $base: IBaseConfig = {
    app: {
        type: 'wlc',
    },
    site: {
        name: 'BestChance',
        url: '',
    },
};
```

# 3.2.2. Конфигурация слоёв

**Layout config** представляет собой объект с конфигурацией сайта по стейтам. Полями объекта являются стейты приложения.
На старте загрузки страницы проверяется нынешний стейт и применяются конкретные настройки. Первое поле **‘app’** –
является структурным, конкретной реализации этого стейта нет, он нужен для указания типового стейта приложения.

## Настройки стейта

Поле             | Интерфейс | Описание
---------------- | --------- | --------
`replaceConfig?` | boolean | Заменить ли дефолтный конфиг из движка, по умолчанию `false`, происходит мерж с дефолтным конфигом;
`subcategories?` | [ILayoutsConfig](../../../interfaces/ILayoutsConfig.html) | представляет собой тонкую настройку стейта с параметрами, по примеру catalog/:category, где `category` – представляет собой параметр, по дефолту отрабатывает с двумя параметрами, `slug` или `category`. Содержимое поля subcategories полностью повторяет `IlayoutsConfig`;
`sections?`      | [ILayoutSectionsConfig](../../../interfaces/ILayoutSectionsConfig.html) | перечислений секций, которые будут представлены на странице, название секции будет фигурировать в классе секции, остальное название произвольно;
`extends?`       | string | Наследовать ли другой стейт, подтянет секции из другого стейта;

Настройки самих секции вынесены в отдельные файлы. Расположены в **src/modules/core/system/config/layouts/sections**.

Файл с настройками секции может включать несколько версий одной секции. Версии секции объединяются через неймспейс в
формате: namespace `<ИМЯ СЕКЦИИ>`

**Пример**

```typescript
export namespace profileContent {
    export const profileDeposit: ILayoutSectionConfig = {
        container: true,
        components: [
            componentLib.wlcDepositWithdraw.deposit,
        ],
    };

    export const profileWithdraw: ILayoutSectionConfig = {
        container: true,
        components: [
            componentLib.wlcDepositWithdraw.withdraw,
        ],
    };
}
```

В основном конфиге, секцию из примера можно подключить так: **sectionsLib.profileContent.profileDeposit** или
**sectionsLib.profileContent.profileWithdraw**.

## Настройка конкретной секции:

Поле             | Интерфейс | Описание
---------------- | ----------| --------
`replaceConfig?` | boolean | Заменить ли дефолтный конфиг из движка, по умолчанию `false`, происходит мерж с дефолтным конфигом;
`container?`     | string or boolean | Применить ли класс `container` для секции, либо конкретный класс;
`components?`    | \([ILayoutComponent](../../../interfaces/ILayoutComponent.html) or string\)\[\] | Массив из объектов компонентов или строк названий компонентов;
`modify?`        | [ILayoutModifyItem](../../../interfaces/ILayoutModifyItem.html)\[\] | Модификация компонента – способ модификации конфига точечно, имеет данные поля, можно добавить, заменить или удалить конкретный компонент или изменить его конфигурацию из секции.
`modifiers?`     | string\[\] | Список модификаторов для секции;
`theme?`         | string | Используемая тема для компонента;
`order?`         | number | Сортировка секции в DOM, по умолчанию все секции имеют вес 1;
`wlcElement?`    | string | Название элемента для автотестов.
`display?`       | IDisplayConfig | Настройка отображения секции, имеет поля: <br> **after** {_number_} -  display component after this window width; <br> **before** {_number_} -  display component before this window width; <br> **mobile** {_boolean_} -  display component only on mobile when true, on desktop when false; <br> **auth** {_boolean_} -  display component only auth user when true, anon user when false;

### Пример modify

```typescript
modify: [
    {
        type: 'replace', // замена компонента
        position: 'base.wlc-logo#2',
        component: {
            name: 'base.test-replace'
        },
    },
    {
        type: 'insert', // вставка компонента
        position: 'before base.wlc-logo#2',
        component: {
            name: 'base.test-insert'
        }
    },
    {
        type: 'delete',  // удаление компонента
        position: 'base.wlc-logo#2',
    },
    {
        type: 'merge', // изменение параметров существующего компонента
        position: 0,
        component: {
            params: {
                wlcElement: 'new-wlc-element',
            },
        },
    },
]
```

Для удаления секции достаточно указать ее в конфиге как `null`

**Пример**

```typescript
'app': {
    sections: {
        footer: null,
    },
},
```

Конфиги компонентов вынесены в отдельные файлы по аналогии с секциями. Расположены в папке
**src/modules/core/system/config/layouts/components** и сгруппированы по модулям.

**Пример**
```typescript

            common: {
                title: '',

            },

    export const active: ILayoutComponent = {
        name: 'bonuses.wlc-bonuses-list',
        params: {
            common: {
                title: '',
                restType: 'active',
                filter: 'active',
            },
        },
    };
    export const inventory: ILayoutComponent = {
        name: 'bonuses.wlc-bonuses-list',
        params: {
            common: {
                title: '',
                filter: 'inventory',
            },
        },
    };
}
```

## Настройка компонента:

Поле              | Интерфейс | Описание
----------------- | ----------| --------
`name`            | string    | Название компонента, содержит в себе два поля, `${moduleName}.${nameComponent}`. Первым в имени компонента идет название модуля, вторым в названии компонента идет имя, экспортируемое из модуля. Пример: 'menu.wlc-menu';
`componentClass?` | string    | Класс компонента (сам инстанс компонента);
`injector?`       | Injector  | Передача инжектора в компонент;
`params?`         | unknown   | Конкретные параметры компонента, в конфиге нужно указывать, чтобы можно было получить все доступные поля. Параметры компонента лучше всего писать подробнее, без сокращений.
`display?`        | IDisplayConfig | Настройка отображения компонента, имеет поля: <br> **after** {_number_} -  display component after this window width; <br> **before** {_number_} -  display component before this window width; <br> **mobile** {_boolean_} -  display component only on mobile when true, on desktop when false; <br> **auth** {_boolean_} -  display component only auth user when true, anon user when false;
`exclude?`        | string\[\] | Исключить компонент из отображения на указанных стейтов;
`include?`        | string\[\] | Отображать компонент только на указанных стейтах;
`reloadOnStateChange?` | boolean | Перезагрузка компонента при изменении стейта;

    },
    'app.profile': {

        },

## Кастомизация PanelLayout

**PanelLayout** - отдельный конфиг для боковых меню. Кастомизируется аналогичным способом. В проекте в папке **config/frontend** необходимо создать файл, который экспортирует переменную `$panelsLayouts` с конфигом (например **05.panels-layouts.config.ts**). По умолчанию файла в проекте нет.

Необходимо убедиться, что конфиг экспортируется в **index.ts**.

```typescript
export {$panelsLayouts} from './05.panels-layouts.config';
```

Поле          | Интерфейс | Описание
------------- | ----------| --------
`showHeader?` | boolean   | Отображать хедер для панели (по умолчанию `true`);
`showClose?`  | boolean   | Отображать кнопку закрытия панели (по умолчанию `true`);
`useScroll?`  | boolean   | Отображать скролл (по умолчанию `true`);
`container?`  | string \| boolean | Применить ли класс `container` для секции, либо конкретный класс;
`display?`    | IDisplayConfig | Настройка отображения компонента, имеет поля: <br> **after** {_number_} -  display component after this window width; <br> **before** {_number_} -  display component before this window width;

**Пример конфига из движка:**
```typescript
export const $panelsLayouts: IPanelsConfig = {
    'app': {
        sections: {
            'left-def': sectionsLib.leftPanel.def,
            'left-v2': sectionsLib.leftPanel.left,
            'left-mobile': sectionsLib.leftPanel.mobile,
            right: sectionsLib.rightPanel.def,
        },
    },
    'app.home': {
        extends: 'app',
    },
};
```

## Переключение типа проекта

Предполагается несколько вариантов лайаутов для разных типов проектов. Например: `wlc` и `affiliate`.

Для быстрого переключения типа проекта нужно в конфиге `$base` проекта в секции **app** указать **type**. В данный момент доступны типы `wlc` и `aff`. По умолчанию устанавливается `wlc`

**Пример**

```typescript
export const $base: IBaseConfig = {
    app: {
        type: 'wlc',
    },
    site: {
        name: 'BestChance',
        url: '',
    },
};
```
