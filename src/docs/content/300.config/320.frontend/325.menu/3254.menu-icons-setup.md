# 3.2.5.4. Setting up menu icons

## Icon sets from gstatic
All default icons are grouped into sets and stored on **gstatic** in the following directories:

* [wlc/icons/v1/]()  
* [wlc/icons/colored/v1/]()
* [wlc/icons/colored/v2/]()
* [wlc/icons/colored/v3/]()  
>
* [wlc/icons/european/v1/]()
>
* [wlc/icons/asian/v1/]()
* [wlc/icons/asiann/categories/v1/]()
* [wlc/icons/asian/categories/colored/v1]()

By default, the menu uses the icon set from [/wlc/icons/european/v1/]()

## Default settings
By default, each menu in the engine already provides own icon settings (the `icons` key): `folder` links to the corresponding folder for search and the full icon search alias looks like `"folder/icon name"`

By default, all menu icons are disabled.

**src/modules/menu/system/config/menu.config.ts**
```typescript
export const menuConfig: IMenuConfig = {
    mainMenu: {
        items: [
            'main-menu:promotions',
            'main-menu:contacts',
        ],
        icons: {
            folder: 'wlc/icons/european/v1',
            use: false,
        },
    },
    categoryMenu: {
        icons: {
            folder: 'wlc/icons/european/v1',
            use: false,
        },
    },
    mobileMenu: {
        items: [
            'mobile-menu:sportsbook',
            'mobile-menu:promotions',
            {
                parent: 'mobile-menu:info',
                items: [
                    'mobile-menu:privacy-policy',
                    'mobile-menu:responsible-game',
                    'mobile-menu:fair-play',
                    'mobile-menu:games-rules',
                    'mobile-menu:terms-and-conditions',
                ],
            },
        ],
        icons: {
            folder: 'wlc/icons/european/v1',
            use: false,
        },
    },
    profileMenu: {
        items: [
            'profile-menu:dashboard',
            'profile-menu:bonuses-as-offers',
            'profile-menu:store',
            'profile-menu:cash-deposit',
            'profile-menu:cash-withdrawal',
            {

                    'profile-menu:bonuses-history',
                    'profile-menu:bets-history',
                    'profile-menu:transaction-history',
                    'profile-menu:tournaments-history',
                ],
            },
            {
                parent: 'profile-menu:account-settings',

                ],

```

## How to enable icons in your project
Go to the **в файл [сonfig/frontend/05.menu.config.ts]()** file in your project (create this file if it is missing) and specify the following key for the corresponding menu: `icons.use = true`, e.g.:
```typescript
import {IMenuConfig} from 'wlc-engine/modules/menu';
export const $menu: IMenuConfig = {
    mainMenu: {
        items: [
            'main-menu:promotions',
            {
                name: gettext('Tournaments'),
                type: 'sref',
                class: 'tournaments',
                icon: 'tournaments',
                params: {
                    state: {
                        name: 'app.home',
                    },
                },
            },
        ],
        icons: {
            use: true,
        }
    },
};
```

For example, let's suppose that you have a main menu item with the `contacts` icon
```typescript
'main-menu:contacts': {
        name: gettext('Contacts'),
        type: 'sref',
        class: 'contacts',
        icon: 'contacts',
        params: {
            state: {
                name: 'app.contacts',
                params: {
                    slug: 'feedback',
                },
            },
        },
    },
```

  In this case, the search will be performed using the [wlc/icons/european/v1/feedback]() alias by default (i.e. folder + icon name).

  If a path to the icon is defined at the project level or at the engine level using the [wlc/icons/european/v1/feedback]() alias, as shown in the example below:

**config/frontend/03.files.config.ts**
```typescript
import {IIndexing} from 'wlc-engine/modules/core/system/interfaces';
export const $localFiles: IIndexing<string> = {
    'wlc/icons/european/v1/feedback': '/icons/feedback.svg'
};
```
then the icon will be included from the project or the engine respectively.

If the icon is missing in the project or engine, the query will be redirected to [gstatic/wlc/icons/european/v1/feedback.svg]()

## How to enable category-specific icons in your project
Go to the [сonfig/frontend/05.menu.config.ts]() (file in your project (create this file if it is missing and then include it in [config/frontend/index.ts]()) and specify the following key: `icons.use = true`, e.g.:
```typescript
import {IMenuConfig} from 'wlc-engine/modules/menu';
export const $menu: IMenuConfig = {
    categoryMenu: {
        icons: {
            use: true,
        }
    },
};
```

By default, all icons will check the [config/frontend/03.files.config.ts]() file for aliases of the following type:
* [wlc/icons/european/v1/jackpots]()
* [wlc/icons/european/v1/popular]()
* [wlc/icons/european/v1/slots]()

If these aliases are present, then icons for the categories will be included from your project.

Otherwise, the queries will be converted into 
* [gstatic/wlc/icons/european/v1/jackpots.svg]()
* [gstatic/wlc/icons/european/v1/popular.svg]()
* [gstatic/wlc/icons/european/v1/slots.svg]()

and the icons will be searched for remotely on **gstatic**

If no icon is found on gstatic, then the system will search for a dummy using the [wlc/icons/european/v1/plug]() alias first and then using 
[gstatic/wlc/icons/european/v1/plug.svg]()

## Categories and custom icons in the mobile menu
In this case, the categories are part of the mobile menu, so you need to check out the mobile menu settings. This menu features the `categoryIcons` key, which can be used for replacing an icon set with another one.

**src/modules/menu/system/config/menu.config.ts** (as set up by default in the engine)
```typescript
import {IMenuConfig} from '../interfaces/menu.interface';
export const menuConfig: IMenuConfig = {
    mobileMenu: {
        items: [
            'mobile-menu:sportsbook',
            'mobile-menu:promotions',
            {
                parent: 'mobile-menu:info',
                items: [
                    'mobile-menu:privacy-policy',
                    'mobile-menu:responsible-game',
                    'mobile-menu:fair-play',
                    'mobile-menu:games-rules',
                    'mobile-menu:terms-and-conditions',
                ],
            },
        ],
        icons: {
            folder: 'wlc/icons/european/v1',
            use: false,
        },
        categoryIcons: {
            folder: 'wlc/icons/european/v1',
            use: false,
        }
    },
```
The default folder for searches is [wlc/icons/european/v1/]()

## Since the icons are unique for the desktop version, you can replace aliases in the following way:
Replace the search folder for another one with a shorter and obviously non-existent path on **gstatic**
```typescript
import {IMenuConfig} from 'wlc-engine/modules/menu';
export const $menu: IMenuConfig = {
    categoryMenu: {
        icons: {
            use: true,
            folder: "icons/categories"
        }
    },
};
```

In the [config/frontend/03.files.config.ts]() file, indicate aliases in the following format: "folder/category slug"
```typescript
import {IIndexing} from 'wlc-engine/modules/core/system/interfaces';
export const $localFiles: IIndexing<string> = {
    'icons/categories/blackjacks': '/icons/categories/blackjacks.svg',
    'icons/categories/casino': '/icons/categories/casino.svg',
    'icons/categories/tablegames': '/icons/categories/tablegames.svg',
    'icons/categories/plug': '/icons/categories/plug.svg',    
    ...
};
```
Now the icons will be searched locally, while aliases of icons will become more neat and convenient.

## Как подключить иконки в проекте
  Переходим в проекте **в файл [сonfig/frontend/05.menu.config.ts]()** (при необходимости его нужно создать)
  Нужно выставить у конкретного меню ключ `icons.use = true`, к примеру:
```typescript
import {IMenuConfig} from 'wlc-engine/modules/menu';
export const $menu: IMenuConfig = {
    mainMenu: {
        items: [
            'main-menu:promotions',
            {
                name: gettext('Tournaments'),
                type: 'sref',
                class: 'tournaments',
                icon: 'tournaments',
                params: {
                    state: {
                        name: 'app.home',
                    },
                },
            },
        ],
        icons: {
            use: true,
        }
    },
};
```

  К примеру есть элемент основного меню с иконкой `contacts`
```typescript
'main-menu:contacts': {
        name: gettext('Contacts'),
        type: 'sref',
        class: 'contacts',
        icon: 'contacts',
        params: {
            state: {
                name: 'app.contacts',
                params: {
                    slug: 'feedback',
                },
            },
        },
    },
```

  В данном случае, по дефолту, поиск произойдет по алиасу [wlc/icons/european/v1/feedback]() (путь к папке + имя иконки).

  Если на уровне проекта или в движке определен путь иконки по алиасу [wlc/icons/european/v1/feedback](), как в примере ниже:

**[config/frontend/03.files.config.ts]()**
```typescript
import {IIndexing} from 'wlc-engine/modules/core/system/interfaces';
export const $localFiles: IIndexing<string> = {
    'wlc/icons/european/v1/feedback': '/icons/feedback.svg'
};
```
  то иконка будет подключена из проекта или движка.

  Если в проекте или движке иконки нету, запрос уйдет на [gstatic/wlc/icons/european/v1/feedback.svg]()

## Как подключить иконки категорий в проекте
  Переходим в проекте в файл [сonfig/frontend/05.menu.config.ts]() (при необходимости его нужно создать и подключить в [config/frontend/index.ts]())
  Нужно выставить ключ `icons.use = true`, к примеру:
```typescript
import {IMenuConfig} from 'wlc-engine/modules/menu';
export const $menu: IMenuConfig = {
    categoryMenu: {
        icons: {
            use: true,
        }
    },
};
```

По дефолту все иконки будут проверять наличие в файле [config/frontend/03.files.config.ts]() алиасов вида:
* [wlc/icons/european/v1/jackpots]()
* [wlc/icons/european/v1/popular]()
* [wlc/icons/european/v1/slots]()

Если они там есть, иконки к категориям будут подключены из проекта.

Если алиасов нет, запросы трансформируются в 
* [gstatic/wlc/icons/european/v1/jackpots.svg]()
* [gstatic/wlc/icons/european/v1/popular.svg]()
* [gstatic/wlc/icons/european/v1/slots.svg]()

и поиск иконок осуществляется удаленно на **gstatic**

Если на gstatic нету иконки, поиск заглушки будет произведен по алиасу
[wlc/icons/european/v1/plug]()
а затем, если не найдено, попробует найти на 
[gstatic/wlc/icons/european/v1/plug.svg]()

## Категории в мобильном меню, другие иконки

**[src/modules/menu/system/config/menu.config.ts]()** (как по дефолту задано в движке)

import {IMenuConfig} from '../interfaces/menu.interface';
export const menuConfig: IMenuConfig = {

  По умолчанию папка для поиска [wlc/icons/european/v1/]()

## На desktop иконки уникальные, как можно поменять алиасы
  Меняем папку для поиска на более короткую и заведомо несуществующую на **gstatic**
```typescript
import {IMenuConfig} from 'wlc-engine/modules/menu';
export const $menu: IMenuConfig = {
    categoryMenu: {
        icons: {
            use: true,
            folder: "icons/categories"
        }
    },
};
```

В файле [config/frontend/03.files.config.ts]() задаем алиасы по принципу `"folder/слаг категории"`
```typescript
import {IIndexing} from 'wlc-engine/modules/core/system/interfaces';
export const $localFiles: IIndexing<string> = {
    'icons/categories/blackjacks': '/icons/categories/blackjacks.svg',
    'icons/categories/casino': '/icons/categories/casino.svg',
    'icons/categories/tablegames': '/icons/categories/tablegames.svg',
    'icons/categories/plug': '/icons/categories/plug.svg',    
    ...
};
```
  Теперь поиск иконок произойдет локально, при этом алиасы иконок более красивые и удобные.
