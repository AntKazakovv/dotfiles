#  3.2.5.2. Menu of categories

## Добавление разделов в основное меню
By default, the project contains two sections: **Casino** and **LiveCasino**. This is set by default in the engine (so you don't need to set up anything on the **fundist**'s side).

## Creating child categories for each section
It is generated automatically. By default, only the **Casino** section can be split into subcategories (so you don't need to set up anything on the **fundist**'s side)

## Creating parent categories
Use the `parents` key to indicate slugs of the categories that you want to display as sections (these are **casino** and **livecasino** sections by default).
These categories will be published in the main menu. By default, the "casino" section will be split into child categories, which will be published in the category menu.
If you remove **casino** from the list, then this section will disappear from the main menu, but will be kept as a section (since **casino** is the default parent category) and its child categories will remain in the category menu. To exclude the **casino** section, see how to exclude categories below.
```typescript
import {IGamesConfig} from '../interfaces/games.interfaces';
 
export const $games: IGamesConfig = {
    categories: {
        parents: [
            'casino',
            'livecasino',
        ],
    },
};
```

### Excluding categories
f you are quite sure that some categories are not needed at all, remove them by default using `exclude.buSlug`. For example, let's exclude the **casino** category:
```typescript
import {IGamesConfig} from '../interfaces/games.interfaces';
 
export const $games: IGamesConfig = {
    categories: {
        exclude: {
            bySlug: [
                'casino',
            ],
        },
    },
};
```
Since only two parent categories (**casino** and **livecasino**) exist in the system, only **livecasino** will be kept in the main menu. At the same time, **livecasino**, just like **casino**, will be split into child categories by default, which will be published in the category menu.  
If no `slug` is indicated for the category, then `menuid` will be used as a `slug`.

## Use cases
**You need only the Livecasino section on your website**
To implement this, exclude the **casino** category via `exclude.bySlug`. Then it will disappear as a category and will be removed from the main menu.  
If you use the `parents` **key**, then the Casino section will disappear from the main menu, but will be kept as a parent category and its child categories will be published in the category menu.
```typescript
import {IGamesConfig} from '../interfaces/games.interfaces';
 
export const $games: IGamesConfig = {
    categories: {
        exclude: {
            bySlug: [
                'casino',
            ],
        },
    },
};
```

#  3.2.5.2. Меню категорий

## Добавление разделов в основное меню
  По умолчанию в проекте создается два раздела: **Casino** и **LiveCasino**.
  Это прописано дефолтными настройками в движке (на стороне **fundist** ничего настраивать не нужно).

## Создание дочерних категорий для каждого раздела
  Формируется автоматически. По умолчанию дробится только раздел **Casino** (на стороне **fundist** ничего настраивать не нужно)

## Создание родительских категорий
  Указываем по ключу `parents` слаги категорий, которые должны быть разделами (по умолчанию это разделы **casino** и **livecasino**).  
  Данные категории будут опубликованы в основном меню. По умолчанию раздел casino раздробится на дочерние категории, которые будут опубликованы в меню категорий.  
  Если из списка убрать **casino**, то из основного меню он будет убран, но продолжит существовать как раздел (так как он по умолчанию категория всегда родительская, это специфическая особенность для **casino**) и его дочерние категории будут в меню категорий. Для того чтобы избавиться от **casino**, см. ниже исключение категорий.
```typescript
import {IGamesConfig} from '../interfaces/games.interfaces';
 
export const $games: IGamesConfig = {
    categories: {
        parents: [
            'casino',
            'livecasino',
        ],
    },
};
```

### Исключение категорий
  Если заранее известно, что некоторые категории вовсе не нужны, их по умолчанию можно убрать используя `exclude.buSlug`.  
  Например, избавимся от категории **casino**:
```typescript
import {IGamesConfig} from '../interfaces/games.interfaces';
 
export const $games: IGamesConfig = {
    categories: {
        exclude: {
            bySlug: [
                'casino',
            ],
        },
    },
};
```
  При этом, так как по умолчанию существовало две родительские категории **casino** и **livecasino**, в основном меню останется только **livecasino**.  
  И **livecasino**, по аналогии с разделом **casino**, будет по умолчанию раздроблен на дочерние категории, которые будут опубликованы в меню категорий.  
  Если у категории отсутствует `slug`, то в качестве `slug` используется `menuid`.

## Практические примеры
**На сайте нужен только раздел Livecasino**
  Для этого нужно исключить категорию **casino** через `exclude.bySlug`. Тогда она перестанет сущестовать как категория и будет убрана из основного меню.   
  При использовании **ключа `parents`** раздел казино был бы убран из основного меню, но при этом бы продолжил существовать как родительская категория и 
  его дочерние категории были бы опубликованы в меню категорий.
```typescript
import {IGamesConfig} from '../interfaces/games.interfaces';
 
export const $games: IGamesConfig = {
    categories: {
        exclude: {
            bySlug: [
                'casino',
            ],
        },
    },
};
```
